May 17, 2026

The Transaction Detail analyzer's Account Mismatch finding got a major refocus. The headline number is now the misposted dollar amount (what needs to move via JE), not the document's net variance — which was $0 for true account mismatches and underplayed the cleanup size. The HOW card walks the analyst through a five-step plan in plain English: it names the relevant DMAAI for the doc type (IM → 3110, IA → 4124, SO/RI → 4240, etc.), points at the row in the DMAAs section to inspect, gives the exact Dr/Cr accounts and amount for the immediate JE, recommends correcting the AAI as the preventative fix, and tells the analyst when to escalate (DMAAs clean → likely a manual JE, not a config issue). Period Mismatch is now a separate pattern with its own resolution path.

May 17, 2026

The Export Analyzer's Transaction Detail template gained a new pattern: "Order line edited between ship-confirm and Sales Update." When a sales order's F4111 cardex captures one quantity but the GL ends up booking a different (higher) quantity because the F4211 line was edited after ship-confirm, the analyzer now diagnoses the post-confirm edit, names the JE to bring the GL into line with cardex, and recommends the JDE controls (Order Activity Rules, Inventory In Hand flag, Security Workbench role restrictions) that prevent recurrence. Preloading a previously-analyzed JDE DMAAI's workbook on the Preload card now also feeds the analyzer's AAI lookups - downstream Transaction Detail diagnoses can now name the resolved GL account from the customer's actual F4095 instead of falling back to a generic placeholder. New customer doc: RR University's "JD Edwards Line Types - Reference" page covers the 11 standard line types, their 8-flag matrix from F40205, and why line type matters when reading a Transaction Detail report.

JDE DMAAI's analyzer gained two new tabs: Pattern Detail (one block per flagged pattern with what / why / step-by-step recommendation and hyperlinked evidence) and Transaction Tracing (T-account flow per module, company, and transaction type for Sales and Inventory, showing where each AAI resolves through the F4095 hierarchy with GL-class branches inline). The transfer-clearing pattern was also corrected to compare AAI 4245 (Inventory in Transit on ST shipments, despite its misleading "A/R Trade" label) against AAI 4320 (Inventory in Transit on OT receipts) per the canonical JDE Distribution AAI convention — the previous 4220/4320 comparison was looking at a sales-side wash pair, producing false positives. * JDE DMAAI's v4: worklist Analysis tab + 4 module tabs + Preload auto-detect

Major iteration that reframes the JDE DMAAI's analyzer template from a categorical findings list into an action-oriented configuration audit worklist, adds Manufacturing and Purchasing module tabs alongside the existing Sales and Inventory tabs, unifies the preload UI behind a single auto-detecting drop zone, and adds a GL class -> Subsidiary discrepancy pattern.

ANALYSIS TAB - reframed as a worklist

The Analysis tab is now a single-table WORKLIST sorted by company. F1, F2, ... label mechanical fixes; Q1, Q2, ... label customer questions. Stable question IDs survive across runs so the analyst's Answer / Decision / Status notes carry over when they re-export F4095 and reload the analyzer.

* Single contiguous worklist table (no per-bucket data break) so Excel's autofilter operates across the whole list. Filter on Type / Module / Company / Status, etc. * 12-column layout: # / ID(hidden) / Type / Module / Company / Task / Doc types / GL classes / Reference / Answer / Decision / Status. The # cell uses a red strip for Fix items, orange for Question items. The ID column is hidden but parsed for carryover. * Reference column cells are HYPERLINK formulas that jump straight to the matching module tab. * Answer column is a data-validation dropdown: Intended / Needs review / Fixed. Status column dropdown: Open / In progress / Done. * Doc types and GL classes show compact lists ("ASGB, BRKR, BUYP (+ 3 more)") in their own columns so the task text stays focused on the AAI pair and the issue. * Worklist items consolidate aggressively to avoid a 3,000-row list - by (co, AAI pair) for net-zero, by (co, AAI) for redundant specifics, etc. * Regression detection: when a prior workbook is loaded and an item marked "Fixed" reappears in the new run, the row is flagged "STILL FLAGGED" with a red strip - the fix didn't stick.

MODULE TABS - 4 modules with collapsible per-transaction sections

Sales, Inventory, Purchasing, and Manufacturing each get their own tab walking the canonical T-account flow per transaction type. Each AAI step shows the resolved Object account split by GL class. Sections default-collapsed so the analyst expands only what they need.

AAI assignments corrected per repo knowledge (AnalysisGuides docs + inventory-distribution-aais.html): * IM (Material Issue) is now Manufacturing: 3110 Cr (Raw Material) / 3120 Dr (WIP). Was incorrectly placed in Inventory with 4122/3120. * IC (WO Completion): 3120 Cr / 3130 Dr. * IH (Labor): 3120 Dr / 3401 Cr. * IS (Scrap): 3120 Cr / 3130 Dr. * IV (WO Variance): 3220 / 3240 / 3260 / 3270 / 3280 - the canonical 32xx range, NOT 3401-3499. * OP receipt: 4310 Dr / 4320 Cr / 4335 Standard Cost Variance (Dr/Cr, receipt-only) / 4337 Material Burden (always Cr, receipt-only) / 4350 Tax. Removed 4330 from OP - that fires at voucher match, not at receipt. * OV voucher match: 4320 Dr / 4330 PPV / 4332 Cost of Sales Variance / 4340 Exchange Rate Variance / 4350 Tax. Removed 4337 from OV (receipt-only). * OD direct ship: + 4365 Supplier Direct Ship Settlement.

Each module also auto-generates "client-defined" sections for any prefix-matching doc type in F4095 not covered by an explicit catalog entry. For Acme that picks up OH / OL / OM / ON / OO on Purchasing and the C2/C3/C5 series on Sales.

PRELOAD CARD - unified drop zone with auto-detect

The Preload card now has a single drop zone that auto-detects three possible DMAAI source formats and routes to the right state store:

* JDE DMAAI's analysis workbook (preferred - full F4095 extract plus the worklist IDs for carryover). State -> PriorWorkbook. * RR Integrity Report 0 (DMAAI Integrity) - for users without databrowser access to F4095. State -> SystemContext. * RR Integrity Report 2 (DMAAI Entry Integrity). State -> SystemContext.

Two status pills (JDE DMAAI's left, RR DMAAI's right) sit in the card head; loading one clears the other (mutex). DMAAITemplate gained a parseReport0 method for the Report 0 column shape (CompanyNumber / TableNumber / DocType / GLClass / BusUnit / Object / Sub) - previously the parser ONLY accepted Report 2's AAIAccount/ModelAccount shape, while the UI label said "Integrity Report 0", which was misleading. Now both work.

NEW PATTERNS

* GL class -> Subsidiary discrepancy detector. For each (co, AAI, GL class), if 80%+ of rows share a dominant sub value but a few rows deviate, flag the exceptions as a customer question. Skips "random" patterns where no sub dominates. * Pattern 7 (transfer clearing) now skips wildcard-tier tuples (gl='****'). Real transactions never use the wildcard as a lookup key, so flagging mismatches at that tier produced false positives. (Acme dropped from 4 findings to 1.) * Removed the speculative "AAI splits across N Object accounts" Q-item path. Splits across GL classes are by design - that's the whole point of GL classes routing to different accounts - so flagging them is a false alarm.

EXCEL-CORRUPTION FIX

The new module tabs use outlineLevel + outlineProperties (for collapsible sections). ExcelJS emits sheetPr children in the wrong child-order (tabColor / pageSetUpPr / outlinePr instead of the OOXML-required tabColor / outlinePr / pageSetUpPr) which Excel rejects as a corrupt file when BOTH pageSetUpPr and outlinePr are present. Workaround: module-tab pageSetup omits fitToPage so pageSetUpPr is never emitted - outlinePr is then the only child and the order is moot. Same workaround applied on the Analysis tab so the HOW TO USE collapsibility works.

• Inventory reconciliation — added a section explaining JDE's own R41500 (batch) and P41500 (interactive) Inventory-to-G/L reconciliation programs and how RR fits alongside them (RR investigates and resolves variances; the JDE programs stamp matched documents as reconciled in F4111 and F0911 after the fact). Added a "batches to confirm with your DBA before reconciling" checklist covering Sales Update, Manufacturing Accounting, Voucher Match / Post, and G/L Post.

• Cycle counts and tag counts — documented what the variance journal from R41413 and R41610 actually contains, and added a warning about the JDE rule that a count entered at a new item-location writes a variance for the entire counted quantity (so a routine count can look like a large shrinkage).

• Item ledger — added a subsection on R41548 (As-Of Updating) explaining why skipping it produces false-positive variances on low-velocity items even when F4111 is correct.

• Item ledger — added a new section on Item Reclassifications (P4116) explaining how paired From / To entries can credit one inventory account and debit a different one, with no purchase or sales activity behind them — the "phantom GL-class movement" pattern in the cardex.

• Inventory costing — added the JDE rule that weighted average and actual cost cannot be used as the sales / inventory cost method for items with stocking type C or R, plus guidance on diagnosing unit-cost movements that show up in the cardex even when on-hand quantity looks fine.
