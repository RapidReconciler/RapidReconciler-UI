What’s new in RapidReconciler.

Newest changes appear at the top — each entry is auto-published from a merge to main. Spot something missing or want context on a fix? rrsupport@getgsi.com.

May 19, 2026

- The four destination pages (RR University, Export Analyzer, Help Desk, Log Analyzer) now share the same welcome-banner template -- wordmark, page pill, Beta, GSI. No more divergent in-banner action buttons or stale topnav formats. "How to use this page" links live in each tool's hero now. - The Export Analyzer's GSI logo is back to the shared 96px standard (was a stale 64px override). - The "Use & analyze" and "Troubleshoot" group labels on the help cover page are now visible (were near-invisible grey on white). - Added the existing "How to find the right answer fast" doc to the RR University's Browse-all-documents drawer (it had been missing from there since the doc was created).

May 19, 2026

- Corrected JD Edwards program references: R30822 (Frozen Cost Update) now replaces R30835 (Cost Simulation) across the analyzer and KB. R30835 is the cost-simulation preview step, not the frozen-update step. - Pattern 5.6 / 5.16 explanations tightened: R30837 (WIP Revaluation) writes both the F4111 "Standard Cost Change" row AND its GL counterpart, so the diagnosis now reads "R30837 partially fired -- GL side never wrote" instead of the older two-program sequencing-gap framing. - OV doc type (PO Receipt -- P4312) now correctly maps to AAI 4310. Was wrongly mapped to 4320 (voucher-match). Pattern 5.4's account-mismatch matrix labels OV docs correctly now. - Work-order Closed status is correctly described as UDC 00/SS (typically 90; customer-defined), not the sales / purchase-order 999. - DMAAI preload persists across browser restarts and reboots -- load once, sticks until you clear. - New Section 10 (JDE Status Code Reference -- WO / SO / PO) and Section 11 (Manufacturing Cost Programs Reference) in the Transaction Detail analyst guide. - New "Doc Type: OP / OV / PV" glossary entry in the PO Receipts reference doc.

May 18, 2026

- Three more Transaction Detail patterns use the cell-based JE-flow matrix in their "How to fix it" card: 5.6 Standard Cost Change, 5.16 Mfg Cost Mismatch, and 5.17 Voucher Variance (standard-cost branch). Matches the format Pattern 5.4 has been using. - JD Edwards cost method (UDC 40/CM) shows up on the "What happened" card for Patterns 5.6 and 5.15 too, plus any future / unmapped pattern. - Pattern 5.17's WHY text is shorter and more direct on both the std-cost and weighted-avg branches. - All bullet cards now have a blank line between bullets — the dense wall-of-text density is gone. - DMAAI preload persists across browser restarts and reboots. Load it once and it sticks; clear it explicitly via the Preload card when you want to switch customers. - New multi-customer F4095 preload script (Treatt and NACoal fixtures land alongside Acme).

May 18, 2026

- Cost method (UDC 40/CM) now also appears on the WHAT card for Pattern 5.6 (Standard Cost Change), Pattern 5.15 (R31802A Orphan), and any future / unmapped patterns — extending the same Ext-column display first added to Pattern 5.16.

May 18, 2026

- Pattern 5.16 (Manufacturing Cost Mismatch) now shows the JD Edwards cost method directly in the WHAT card ("Cost method (F4111 Ext): 07 (Standard Cost)") and uses the method to sharpen the WHY card's root-cause framing — standard, manufacturing-last, and other actual-costing regimes each get the right description instead of a generic "frozen vs actual" line.

May 18, 2026

- Two new Transaction Detail patterns: **Manufacturing Cost Mismatch** (catches frozen-standard vs cost-component drift on work-order completions) and **Voucher Variance on Inventory** (diagnoses PV vouchers whose variance landed on the inventory account, branching on standard vs weighted-average cost). - Pattern 5.4 (Account Mismatch) and Pattern 5.14 (R31802A Orphan) now surface as co-occurring "Also detected" findings rather than gating each other out. - Every Transaction Detail analysis tab now shares the same bullet-style output — WHAT facts, WHY root causes, HOW action steps — replacing the older Step 1 / Step 2 / Step 3 prose layout. - F4111 rows now carry the JD Edwards cost method directly. The analyzer reads the value rather than inferring it from unit-cost arithmetic. - Expanded JD Edwards Cost Methods reference in RR University — full UDC 40/CM table (15 methods), manufacturing-relevance notes for 07 / 02 / 09, and Oracle's guidance on mixing methods. - New "Voucher Variance on Inventory" section in the Transaction Detail Analysis Guide for the corresponding analyzer pattern.

May 18, 2026

- Account Mismatch reports now show the corrective journal entry as a four-scenario flow across the three accounts touched (what was supposed to happen / what actually happened / corrective entry / end result). Credits render in red parentheses, debits in black; the bold End Result row confirms the books reconcile in one glance. - Help portal cards reorganized into two groups: "Use & analyze" (University + Export Analyzer) and "Troubleshoot" (Help Desk + Log Analyzer). - New Log Analyzer card on the help portal — drop a browser console dump or an RR agent log file, the tool flags the lines that actually need attention. - Log Analyzer page now uses the standard Help Desk header and footer, has a "How to use this tool" link, and larger input cards for easier reading. - 90-second tour now runs end-to-end across all four destination pages (was breaking mid-tour). - Help Desk "What to include for faster help" drawer is expanded by default. - GSI logo enlarged across the help portal banner so the "All signal. No noise." tagline reads comfortably. - Inventory Distribution AAIs reference: removed references to Flexible Accounting on the purchasing side (not a JDE feature), expanded the segment-widths table to cover all flex-eligible segments, and added an inventory cross-DC cost-standardization use case.

May 18, 2026

Account Mismatch analysis now shows a T-account picture of the corrective journal entry. The JE direction is derived from what should have posted for the document type (not from the F4111 model account), so the entry correctly reverses the bad post and applies the entry that should have happened. Investigation hand- holding has been dropped — the JE is the fix.

May 18, 2026

Transaction Detail analyzer output is now substantially tighter and more actionable. A new pattern (R31802A Orphan Cardex Row) detects the manufacturing case where one F4111 row on an IC / IM / IH / IS doc has no matching F0911 entry — common when an R31802A run is interrupted, a cost rollup is incomplete, or the row's "already processed" flag was set out-of-band. The recommended JE names the actual offsetting account (or "Work in Process" when not resolvable) so the analyst can post without leaving the analysis tab. The boilerplate "Verify before acting" disclaimer that repeated on every report has been folded into a one-line footer caveat. The Evidence section is gone — root-cause rows are highlighted directly on the Transaction Details sheet. WHAT and WHY cards are now bullet lists rather than prose paragraphs. The Period Mismatch detector was also fixed — it no longer mis-fires on docs where the RR Summary has multiple rows on the same account but same period (orphan-row docs were hitting it incorrectly).

May 17, 2026

The new "JD Edwards Account Structure" reference doc was rewritten with fictional account numbers throughout (5000.140000, 5000.140100.RM1, 9000.501000.10) so it teaches the same BU.OBJ.SUB parsing skill without leaking any customer's actual chart of accounts. The two recent additions (JDE Line Types, JD Edwards Account Structure) also now show up where users actually browse — under Inventory in the RR University drawer and the Help Desk drawer's Inventory section.
