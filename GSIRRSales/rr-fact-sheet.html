How it works
3-Point Reconciliation Across the Supply Chain
01  ·  Inventory Reconciliation

Inventory ledger to GL, automated. Variance is split into six named sources — Carry Forward, GL Batches, End of Day, Transactions, Cardex, Manual JEs — so each one points back to the action that fixes it. Includes the As-Of page for period-end position by item, branch, and lot. (Compares F4111 to F0902.)

02  ·  In-Transit Reconciliation

Shipments and receipts balanced to the GL. The exclusion process isolates closed-order residuals and surfaces the exact journal entry needed to clear them — no more carrying in-transit variance forward. (Links ST/OT order pairs.)

03  ·  PO Receipts (RNV)

Received-not-vouchered, reconciled. Shows only unreconciled orders, not the entire receipt history, with built-in RNV aging by period and document-level analysis with audit notes. (Matches F43121 to F0911.)

Proactive Monitoring
Eleven configuration checks — silently working in the background
A single misconfigured AAI silently routes every transaction of that type to the wrong account. RapidReconciler runs 11 JD Edwards configuration checks every month — covering AAI mappings, GL classes, UOM conversions, frozen cost, sales duplicates, cutoff timing, and more — so misconfigurations get fixed before they cause reconciling items.
