## Overview

A Zero Balance Adjustment is automatically created in the General Ledger when the inventory quantity for an item goes to zero while a cost value remains associated with that item. This occurs regardless of which inventory cost method is selected in the Cost Revisions file (F4105).

The underlying principle is that **an inventory value cannot exist if there is no quantity on hand**. When this condition is detected, the system automatically generates a journal entry to bring the inventory value to exactly zero.

Zero Balance Adjustments apply to the **Purchasing** and **Inventory** modules only. The Sales system was not designed to use this functionality.

---

## Section 1: Why Zero Balance Adjustments Occur

A Zero Balance Adjustment is triggered when two conditions are both true after a transaction is processed:

1. The inventory **quantity on hand reaches zero** (or crosses through zero).
2. A **positive or negative cost value** still remains associated with the item after the transaction.

The most common cause is a **cost difference** -- the cost at which inventory was removed differs from the cost stored in the F4105 Cost Revisions file prior to the transaction. This cost difference leaves a residual dollar value that cannot remain in the inventory account with no quantity to support it.

## Section 2: Purchasing Zero Balance Adjustments

### 2.1 DMAAI Configuration

Zero Balance Adjustments in the Purchasing module are controlled by the following AAIs in the Distribution/Manufacturing AAI Values table (F4095):

| AAI | Account | Direction | Document Type |
|---|---|---|---|
| **4400** | Inventory | Debit | IB (default) |
| **4405** | COGS or Expense | Credit | IB (default) |

These AAIs must be configured for the correct company, document type, and GL class code combination. If AAI 4400 or 4405 is missing or points to an incorrect account, the zero balance adjustment will either fail or post to the wrong GL account, creating a reconciliation variance.

For full AAI configuration guidance, see the [DMAAI Reference Guide](../MDS/dmaai-reference-guide.md).

### 2.2 Program

Zero Balance Adjustments in Purchasing are generated by:

- **P4312** -- Enter Receipts by Purchase Order or by Item

### 2.3 Transactions That Trigger a Purchasing Zero Balance Adjustment

| Transaction | Description | Trigger Condition |
|---|---|---|
| **Negative quantity PO receipt** | A purchase order receipt entered with a negative quantity that drives on-hand to zero | Quantity reaches zero and a residual value remains after the receipt cost is applied |
| **Receipt reversal** | A reversal of a previously posted receipt that drives on-hand to zero | Quantity reaches zero and a residual value remains after the reversal |

### 2.4 Transactions That Do NOT Trigger a Purchasing Zero Balance Adjustment

| Transaction | Reason |
|---|---|
| **Standard positive receipt (P4312)** | Adds quantity -- cannot drive on-hand to zero |
| **Voucher match (P4314 / P0411)** | Does not affect inventory quantity; only settles the RNV liability |
| **Purchase order entry (P4311)** | No inventory quantity or value impact at order entry |
| **Landed cost (P43214)** | Adjusts cost only; does not affect on-hand quantity |
| **Receipt routing (P43250)** | Moves inventory through routing steps; does not reduce on-hand to zero through this program |

### 2.5 Purchasing Example

**Starting position:**

- Quantity on Hand: **25**
- Unit Cost (cost method 02 -- Weighted Average): **$5.00**
- Total Inventory Value: **$125.00** (25 x $5.00)

**Transaction:** A purchase order receipt for a quantity of **-25** at **$4.00 each**.

| | Quantity on Hand | Value |
|---|---|---|
| Before the transaction | 25 | $125.00 |
| Transaction amount | -25 | -$100.00 |
| After the transaction | 0 | **$25.00** |

After the transaction, quantity is zero but **$25.00 of value remains**. The system generates a zero balance adjustment to eliminate the residual value.

**Journal entries -- Receipt:**

| Entry | Debit | Credit | AAI |
|---|---|---|---|
| Inventory | -$100.00 | | 4310 |
| Received Not Vouchered | | $100.00 | 4320 |

**Journal entries -- Zero Balance Adjustment:**

| Entry | Debit | Credit | AAI |
|---|---|---|---|
| Inventory | -$25.00 | | 4400 |
| COGS or Expense | | $25.00 | 4405 |

The zero balance adjustment record in the Item Ledger (F4111) will display **"Zero Balance Adjustment"** in the Explanation field.

## Section 3: Inventory Zero Balance Adjustments

### 3.1 DMAAI Configuration

Zero Balance Adjustments in the Inventory module are controlled by the following AAIs in F4095:

| AAI | Account | Direction | Document Type |
|---|---|---|---|
| **4126** | Inventory | Credit | IB (default) |
| **4128** | COGS or Expense | Debit | IB (default) |

As with Purchasing, these AAIs must be correctly configured for the applicable company, document type, and GL class code. Missing or incorrectly mapped AAIs will result in posting errors or reconciliation variances.

For full AAI configuration guidance, see the [DMAAI Reference Guide](../MDS/dmaai-reference-guide.md).

### 3.2 Programs

Zero Balance Adjustments in Inventory are generated by:

- **P4112** -- Inventory Issues
- **P4113** -- Inventory Transfers
- **P4114** -- Inventory Adjustments

### 3.3 Transactions That Trigger an Inventory Zero Balance Adjustment

| Transaction | Program | Trigger Condition |
|---|---|---|
| **Inventory issue** | P4112 | Issue drives on-hand to zero and a residual cost value remains |
| **Inventory transfer** | P4113 | Transfer drives on-hand to zero at the sending location and a residual cost value remains |
| **Negative inventory adjustment** | P4114 | Adjustment drives on-hand to zero and a residual cost value remains |

### 3.4 Transactions That Do NOT Trigger an Inventory Zero Balance Adjustment

| Transaction | Program | Reason |
|---|---|---|
| **Positive inventory adjustment** | P4114 | Adds quantity -- cannot drive on-hand to zero |
| **Inventory reclassification** | P4116 | Reclassifies lot or location data; does not affect on-hand quantity or value in a way that triggers a zero balance adjustment |
| **Cycle count update** | R41413 | Uses AAIs 4152/4154; zero balance adjustment AAIs 4126/4128 are not invoked |
| **Tag count update** | R41610 | Uses AAIs 4152/4154; zero balance adjustment AAIs 4126/4128 are not invoked |
| **Future cost update** | R41052 | Uses AAIs 4172/4174; zero balance adjustment AAIs are not invoked |
| **Cost revision change (P4105)** | P4105 | Creates an IB transaction directly; does not go through the zero balance adjustment process |
| **Sales shipment** | P4205 / R42800 | Handled by Sales AAIs 4210/4220; zero balance adjustment AAIs not invoked in the Sales module |
| **Manufacturing completions** | R31802A | Handled by Manufacturing AAIs 3120/3130; zero balance adjustment AAIs not invoked |

### 3.5 Inventory Example

**Starting position:**

- Quantity on Hand: **30**
- Unit Cost (cost method 02 -- Weighted Average): **$10.00**
- Total Inventory Value: **$300.00** (30 x $10.00)

**Transaction:** An inventory issue (P4112) for all 30 units at a unit cost of **$12.00 each**.

| | Quantity on Hand | Value |
|---|---|---|
| Before the transaction | 30 | $300.00 |
| Transaction amount | -30 | -$360.00 |
| After the transaction | 0 | **-$60.00** |

After the transaction, quantity is zero but a **negative value of -$60.00 remains**. The system generates a zero balance adjustment to bring the value back to zero.

**Journal entries -- Inventory Issue:**

| Entry | Debit | Credit | AAI |
|---|---|---|---|
| Expense or COGS | $360.00 | | 4124 |
| Inventory | | -$360.00 | 4122 |

**Journal entries -- Zero Balance Adjustment:**

| Entry | Debit | Credit | AAI |
|---|---|---|---|
| Expense or COGS | $60.00 | | 4128 |
| Inventory | | -$60.00 | 4126 |

## Section 4: Sales Module

The Sales system was **not designed** to use Zero Balance Adjustments. Sales transactions are processed through Sales AAIs 4210 and 4220 (inventory and COGS), and the zero balance adjustment AAIs (4126/4128, 4400/4405) are not invoked by any Sales program.

If a zero balance adjustment appears to be needed in the context of a sales transaction, the root cause should be investigated in the Inventory or Purchasing modules where the on-hand quantity was last affected.

---

## Section 5: Identifying Zero Balance Adjustments in the Item Ledger

Zero Balance Adjustments appear in the Item Ledger (F4111) and can be identified by the following characteristics:

| Field | Value |
|---|---|
| **Document Type** | IB |
| **Explanation field** | "Zero Balance Adjustment" |
| **Quantity** | Zero (the adjustment affects value only, not quantity) |
| **Extended Cost** | The residual dollar amount being cleared |

When reviewing the cardex in Excel, zero balance adjustments should be included in the extended cost summary but excluded from quantity totals (since their quantity is zero). They do not carry a posting code of "X" and are not memo transactions -- they represent actual GL postings.

## Section 6: Reconciliation Considerations

Zero Balance Adjustments post to AAIs 4126/4128 or 4400/4405 depending on the module. If these AAIs are configured to point to non-inventory accounts (such as COGS or expense accounts), the adjustments will not appear in the inventory GL account balance, potentially creating a discrepancy between the Stock Status report and the Trial Balance.

> **Best Practice:** Configure AAIs 4400 and 4126 to point to the same inventory account used for standard inventory transactions (AAIs 4310 and 4122 respectively). This ensures that zero balance adjustments remain within the inventory account and do not create unexplained variances.

If zero balance adjustments are posting to expense or COGS accounts and creating reconciliation variances, review the AAI configuration for 4405 and 4128 and consider whether the offset account is appropriate for your organization's financial reporting requirements.

For guidance on the broader reconciliation context, see the [Stock Status and Trial Balance Reconciliation](../MDS/stock-status-trial-balance.md) guide.
