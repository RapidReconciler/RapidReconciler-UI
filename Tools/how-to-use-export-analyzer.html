How-to guide

How to use the Export Analyzer.

Drop a RapidReconciler .xlsx export, confirm the template, download a worked-up analysis workbook with findings and a recommended next step — about a minute, all in your browser.

The Export Analyzer takes any of eight supported RapidReconciler exports and produces a formatted analysis workbook with a priority-colored variance card, an evidence list, and a WHAT / WHY / HOW recommendation. Three steps on one page: Drop, Confirm, Review & download.

Part 1What it does (and what it doesn’t)

The analyzer is a hypothesis generator, not a verdict. It surfaces likely root causes based on a pattern library that grows with every customer report it sees. Customer-specific DMAAI conventions, order-type splits, and uncommon transaction types vary, so a finding that looks confident in the workbook should still be verified against your JDE setup before you post a journal entry or change configuration.

Drop your export

One file, one analysis

Drag an .xlsx RapidReconciler export onto Step 1. The template is detected automatically; you confirm or override in Step 2.

Verify in JDE before acting

Check before you post

Every finding is a hypothesis. Open the cited rows in JDE and confirm the diagnosis before you post a JE or change configuration.

Nothing leaves your machine. The analyzer is a single HTML page; every byte of your export is parsed in the browser tab you have open. No data is uploaded to GSI, no file is stored on a server. Close the tab and the analysis is gone.

Part 2Drop your export

Step 1 on the analyzer page has a drop zone. Two ways to load a file:

  • Drag and drop an .xlsx file onto the green zone.
  • Click the zone to open a file picker and choose one.

The analyzer accepts any of the eight supported RapidReconciler exports. They come from three places in the app:

  • Inventory Integrity page — DMAAI Entry Integrity (Integrity Report 2), GL Class Integrity (Integrity Report 5), and Frozen Cost Integrity (Integrity Report 6). Pick from the Select Report dropdown, then click Export.
  • Inventory Reconciliation page — Unposted GL Batches, End of Day Variance, and Account Roll Forward. Open the page for the period, then click into the relevant pop-up.
  • Row-expansion exports — Transaction Detail from the Inventory Transactions page, and Cardex Variance from the Inventory As Of page. Click the + on a row to expand its details, then click Export.

The Supported reports widget below the three cards on the analyzer page shows the full list with the export location for each — open it any time you’re not sure where a report lives.

Detection is automatic. The analyzer fingerprints your file by sheet name plus column headers and picks one of the eight templates. You don’t need to tell it which export you dropped — if the detection is wrong, the manual override appears in Step 2.

Part 3Confirm the template

Once the file is loaded, Step 2 shows one of three states under the file name:

  • Detected: X (87% match) — high confidence. The analyzer locked the template and moved on to Step 3 automatically. Click Change template only if you know it picked wrong.
  • Likely: X — confirm below — lower confidence. A grid of all eight templates appears. The likely match is highlighted; click it to confirm, or pick a different one if you know better.
  • No confident match — the fingerprint didn’t resolve. The full template grid appears; pick the one that matches what you exported.

Each template card in the grid names what it’s built for. Match it against the report you exported and click. Step 3 unlocks once Step 2 is confirmed.

Pick the right template if you override. The analysis assumes the columns and sheet shape of the selected template — pointing it at a file that doesn’t match will either fail to parse or produce a misleading variance card. When in doubt, re-export the report you intended and let detection run again.

Part 4(Optional) Load Integrity Report 0 for prescriptive output

At the top-right of the Step 1 card there’s a small DMAAI status pill. Below the drop zone there’s an Optional context section with a Load Integrity Report 0 button. This is the only template-agnostic input on the page and it’s entirely optional.

What it does

Loading Integrity Report 0 (DMAAI Integrity) from RapidReconciler lets the analyzer name the actual GL accounts your AAIs resolve to in the diagnosis. Without it, the Transaction Detail Pattern 5.6 finding falls back to generic placeholders — for example, “credit AAI 3240 (Material Variance) or 3260 (Planned Variance).” With Integrity Report 0 loaded, the same diagnosis names the GL account each AAI resolves to for your company and GL class.

If neither AAI 3240 nor 3260 is configured for the relevant company / GL class, the diagnosis flips to “configuration gap — configure the AAI first” instead of recommending a posting. That’s a more actionable finding than the generic placeholder, which is why loading the report is worth a few seconds when the template is Transaction Detail.

When to bother

The other seven templates don’t use the integrity report at all. So:

  • Skip it for Cardex Variance, GL Class Integrity, DMAAI, Frozen Cost Integrity, Unposted GL Batches, End of Day, and Account Roll Forward. Loading it doesn’t hurt — it just doesn’t change the output.
  • Load it once if you’re working through several Transaction Detail exports in a session. The status pill turns green once the report is loaded; the analyzer remembers it for the rest of the tab’s lifetime.
  • Session-only. Closing the browser tab clears the loaded report. The next session starts with the pill off.
Where to export Integrity Report 0. Same place as the other integrity reports — open the Inventory Integrity page in RapidReconciler, pick Integrity Report 0 (DMAAI Integrity) from the Select Report dropdown, and click Export. Then click the analyzer’s Load Integrity Report 0 button to load it.

Part 5Review and download

When Step 2 is confirmed the analyzer runs and Step 3 unlocks. You get a quick inline preview — the variance card and the headline finding — with a Download workbook button. The downloaded .xlsx has three tabs:

Analysis tab

The presentation tab. Read top-to-bottom:

  • Branded header strip — the template name plus a sub-line of context (item number and branch for Cardex, GL class and account for GL Class Integrity, etc.).
  • “Verify before acting” disclaimer — the same caveat the page banner shows. Treat the finding as a hypothesis.
  • Variance card — the headline number, colored by priority:
P1

Investigate immediately

The variance is at least 50% of the document amount — large enough relative to the underlying transaction that it warrants action now.

P2

Review within 1 business day

The variance is between 10% and 50% of the document amount — worth a look but not blocking.

P3

Low priority — include in next backlog

The variance is under 10% of the document amount — small enough that it can wait for your next housekeeping pass.

The priority is computed purely from the variance-to-document-amount ratio, not from a guess at severity — a $1 variance on a $1 line is still P1, and a $1,000 variance on a $50,000 line is P2. That keeps the bucketing consistent across documents and templates; the dollar magnitude is shown alongside the priority so you can sanity-check both at once.

  • WHAT / WHY / HOW cards — three side-by-side cards that name the root cause (WHAT), explain why the analyzer flagged it (WHY), and propose a specific resolution (HOW). The HOW card is where you’ll find the JE recommendation, the configuration change, or the “no action needed” verdict.
  • Evidence list — the specific rows on the Source data sheet the analyzer keyed off. Each entry links back to the source row so you can see exactly what triggered the finding.

Source data sheet

Your original export, preserved verbatim. The analyzer doesn’t modify it — the sheet is there so the Evidence list has a target to link to, and so anyone reviewing the workbook can see the raw data the analysis is based on. If you re-share the workbook with a teammate, they can audit the analysis without needing to re-export.

Notes / Feedback tab

A structured form at the back of the workbook. Prompts include:

  • Was the diagnosis correct? Yes / Partially / No.
  • What’s the real root cause? A free-text box for the answer when the analyzer missed it.
  • Customer-specific context. DMAAI conventions, order-type splits, anything that would have helped the analyzer reach the right conclusion.

At the bottom of the tab is a one-click Email this feedback to RR Support link that opens your default mail client pre-addressed to rrsupport@getgsi.com. Attach the workbook and send.

Part 6When a finding doesn’t fit, send feedback

Patterns are added to the library over time, so the analyzer doesn’t recognize every situation yet. When a finding misses the mark:

  1. Open the Notes tab in the downloaded workbook.
  2. Fill in the prompts. “Partially” or “No” on the first prompt, then the real root cause in the free-text box. Mention any customer-specific context that would have changed the diagnosis.
  3. Save the workbook with your notes included.
  4. Click the email link. Attach the saved workbook in your mail client and send.

You can also send feedback without filling in the workbook — the analyzer page’s About these analyses band has a direct rrsupport@getgsi.com link. Either way, your workbook (or your description) is the only thing leaving your machine, and only through your own email client. Nothing flows to a GSI server through the analyzer itself.

Why feedback matters. Every misfire that gets reported expands the pattern library. The next time someone runs an export with the same fingerprint, the analyzer catches the case you flagged. The supported template list and the diagnostic patterns inside each template both grow this way.

Next stepsRelated tools and references

The analyzer is one of three customer-facing surfaces in the RapidReconciler help portal:

  • Help Desk — symptom-based runbook search. Use it when the analyzer isn’t the right tool — the issue is in the app, the agent, or the network rather than in a posted export.
  • RR University — Inventory module — the underlying JDE concepts the analyzer reasons over: cardex variance, DMAAI, GL class integrity, weighted average cost, and the rest. Worth a read when a finding mentions a concept you want to look up.
