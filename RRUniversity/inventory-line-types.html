JD Edwards Line Types — Reference

The single column on a sales or purchase order that determines whether the line touches inventory, posts to the GL, allows backorders, prints on documents, and a half-dozen other behaviors. Line type is read from F40205 and applied to every sales-order and purchase-order line.

Overview

Every line on a sales order or purchase order in JD Edwards carries a Line Type code (the LineTy column in most reports). The line type is what tells JDE whether the line writes to the item ledger, whether it posts to the GL, whether it can be backordered, and several other behaviors. The line-type constants are stored in table F40205 (Order Line Type Constants) and are usually configured once at implementation and then left alone.

Line type drives behavior, not value. The amount and account on a line still flow through the standard pricing and DMAAI configuration — line type just controls whether each downstream step fires.

Why this matters for RapidReconciler: two reconciliation patterns key off line type directly. Mixed line types on a return doc — when stock and non-stock lines share an order type, DMAAI routes both through the same AAI and the non-stock dollars-only lines post to the inventory account by mistake. Post-ship-confirm order edits — the variance only shows up on stock lines because non-stock lines don't write F4111 at confirm time. Both are surfaced by the Transaction Detail template in the Export Analyzer.

Field Meanings

FieldWhat "Y" means
Inventory InterfaceThe line updates the Item Ledger (F4111) and Item Location (F41021) when the order is processed. Only Y when there's a real inventory movement to record. Verifiable from a Transaction Detail report: a line with Inventory Interface = Y should have at least one F4111 row (often several, one per lot picked); a line with Inventory Interface = N should have no F4111 row at all. A missing F4111 row on a Y-line that's been ship-confirmed, or an F4111 row on an N-line, both warrant investigation.
GL InterfaceThe line creates journal entries in F0911. Most line types are Y here — T (text only) is the notable exception.
Commitment ReliefThe line relieves commitments in F43199 (PO side) or the equivalent sales-side records when received or closed. N on freight / recurring charges — those don't commit inventory.
Print on DocumentsThe line appears on printed sales orders, purchase orders, pick slips, and invoices.
Update Sales/PO HistoryThe line updates F42119 (sales history) or F43199 (purchasing history) when the order completes.
Update Open OrdersThe line updates the open-order tracking files so the order shows on open-PO and open-SO reports.
Backorder AllowedIf the line can't be fulfilled in full, JDE allows the unfulfilled portion to be backordered. Only Y on the line types that physically pull from inventory (S, W, B).
Partial Shipments AllowedThe line can be shipped in multiple installments rather than all at once. Same pattern as Backorder — tied to real inventory line types.

Per-Line-Type Notes

SStock item

The default for any line that moves real inventory. Writes F4111 at ship-confirm, posts to the inventory and COGS accounts via the standard sales / inventory DMAAI tables (4220 / 4240 for sales, 4122 / 4124 for inventory adjustments, etc.). Allows backorders and partial shipments. This is the line type that most reconciliation patterns assume.

NNon-stock item

For items you sell or purchase that aren't tracked in inventory — service contracts, drop-ship items billed through a third party, intangibles. GL Interface is Y but Inventory Interface is N, so the line posts dollars to the GL without touching the item ledger. The DMAAI route for the order type still fires; if it routes through the inventory account, the non-stock line's dollars land on the inventory account by mistake. This is the classic mixed-line-types-on-a-return-doc symptom — if a return order carries both S and N lines and posts a variance, the non-stock line is almost always the cause.

JJob cost / service

Same posting shape as N — dollars-only, no inventory hit — but tied into Job Cost accounting so the GL posting routes against the job number rather than the standard sales / inventory accounts. Common in project-based businesses (construction, services).

FFreight

Captures freight charges as a separate line on the order. Commitment Relief is N — freight doesn't commit inventory, so when the order closes there's nothing to relieve. The GL hit lands on a freight expense / revenue account configured in DMAAI 4280 (sales freight) or 4385 (PO freight).

TText only

Pure documentation — instructions to the picker, customer notes, line descriptions. No inventory, no GL, no history. The only Y is "print on documents." If you see a T line driving accounting questions, something is wrong — text lines should never produce a variance.

MMiscellaneous charge

For one-off charges that aren't freight, aren't tax, and aren't tied to a stock item — restocking fees, expedite fees, custom-cutting charges, setup fees. Dollars-only with commitment relief on, so the charge tracks through the order lifecycle and clears properly at close.

WWork order / manufacturing

Used on sales orders that trigger a make-to-order work order. Behaves like S from the order's perspective (writes F4111, allows backorders, etc.) but the inventory movement is tied to the work order rather than a direct pick from stock. Most relevant for shops that ship from finished-goods inventory created by a same-day work order.

BBulk stock

For items shipped in bulk (liquids, granular product, anything sold by weight or volume rather than countable units). Same flags as S but the supporting bulk-product features (in-line gain/loss accounting, bulk-product status updates) fire on this line type.

DDirect ship

The supplier ships directly to your customer; the goods never enter your inventory. Inventory Interface is N, but the GL Interface is Y so revenue and the corresponding cost both post normally. JDE creates a paired PO and SO that close together when the supplier confirms shipment. See AAI 4365 (Supplier Direct Ship Settlement) for the OD doc-type accounting.

OOutside operations

Used on work orders for outside processing steps — sending an in-process item to a vendor for plating, heat treatment, etc., then receiving it back into the work order. Dollars-only because the inventory side of the movement is handled by the work order routing.

RRecurring charges

For lines that should re-bill on a schedule — maintenance contracts, subscription fees, periodic service charges. Commitment Relief is N because the charge never closes in the usual order-close sense; it lives across periods.

Why Line Type Matters for Reconciliation

Three reconciliation patterns are sensitive to line type directly. If a variance shows up on one of these, line type is the first thing to check.

  1. Mixed line types on a return / credit doc. DMAAI configuration routes by order type and GL class, not by line type. When a return (RI / RM / CR / RE / SR) carries both stock (S) and non-stock (N, M, etc.) lines, every line on that order posts through the same DMAAI route — correct for the stock returns, wrong for the dollars-only credits. Resolution is usually a systemic order-type split (separate order types for stock returns vs. dollars-only credits).
  2. Post-ship-confirm order edits. Only stock-family line types (S, W, B) write F4111 at ship-confirm. If someone edits the line qty between ship-confirm and Sales Update (R42800), the cardex stays at the confirmed qty while the GL gets the post-edit qty — producing a variance equal to (new qty − confirmed qty) × unit cost. The patch is process / access-control, not configuration — tighten Order Activity Rules so the post-confirm status is terminal-editable, confirm the Inventory In Hand flag is being set, and restrict P4210 / P42101 edit access to roles that need it.
  3. Non-stock line on a sales order producing an inventory-account GL hit. When a single sales order mixes line types and the DMAAI for the order type points at an inventory account, the non-stock dollars land on inventory — same root cause as #1, but you may see it on a "clean" non-return order if the non-stock line is the only oddity.

If you can read the LineTy column on the Orders section of the Transaction Detail export, you can usually tell at a glance whether one of these patterns is in play.

Exporting Your Own F40205

Customer environments often customize one or more flags on the standard line types or add custom line types for specific business processes. To compare your actual setup against the vanilla defaults above:

  1. Open JDE Database Browser (program P964001).
  2. Query table F40205 (Order Line Type Constants).
  3. Export the result to Excel.
  4. Compare each row's flag values against the table above. Customizations are the rows that don't match the vanilla defaults — document them.

Custom line types are common. Many JDE shops add line types for industry-specific use cases — a P for prepaid items, a Q for quote-only lines, a K for kit components, and so on. Custom line types still have an entry in F40205; the flags they carry determine how they behave. If you find a line type in the export that isn't covered above, look up the flags in F40205 and reason from there.
