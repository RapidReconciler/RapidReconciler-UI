Export does nothing when you click it

You click Export to Excel or a download button and nothing happens — no file, no error message, no spinning indicator. The RR service has typically drifted into a state where the export call silently fails. Clicking the in-UI Restart Service button resets it.

Quick fix

  1. Locate the Restart Service button in the RapidReconciler UI and click it.
  2. Wait for the restart to complete.
  3. Retry the export on the same report.

If the button isn’t visible to you

The Restart Service button isn’t available to all users. If you don’t see it, contact your RR Administrator — a power user within your organization — and ask them to either click it themselves or grant you access to that function.

If it keeps recurring

One restart isn’t a problem. A pattern of needing it daily or weekly is — the pattern itself is the diagnostic clue. Capture:

  • Time since the last Restart Service.
  • How often the issue recurs (next day, every few days, after long idle).
  • Whether other users at your site hit it at the same time.

The full triage path — from the in-UI Restart Service button through to the IT-side full application-server reboot when the workaround stops holding — is in the Excel-export-failures scenario.

!

If Restart Service doesn’t fix it, multiple users hit it daily, or new symptoms appear, see Email RR Support.

Report won’t run

You click to generate a report and it hangs, returns an error, or never produces output. Several causes — the System Status light is the fastest way to narrow it down.

First, check the status lights

At the top of every reconciliation page is a module status bar with two indicators:

  • System Status — if this is red, the back-end job has crashed and reports won’t complete until it’s recovered. Click the red indicator to export the crash details and email them to support. See the system-status-red scenario.
  • Inventory Validation — if this is red, the period roll-forward failed (typically an unposted GL batch in the prior period). Reconciliation reports may still run but figures will reflect the variance. See the inventory-validation-red scenario.

If status lights are green

  1. Check your filters — an overly restrictive filter or empty date range can produce a no-results report that looks like a hang.
  2. Narrow the date range or scope — very large data sets can time out. Try the same report against a single period or a single business unit to see if it completes.
  3. Hard-reload the page with Shift+F5 and try again — clears any cached state from a previous run.
  4. Confirm you have permission for the report. Some reports are scoped to specific user roles. If you used to be able to run it and can’t now, your RR Administrator may have changed your permissions in Admin → Users.
  5. Try a different report. If only one report fails but others work, the issue is in that report’s data — not your environment.
!

If the status lights are green and the steps above don’t help, see Email RR Support with the report name, the filters you used, and a screenshot.

Export is empty or wrong

The export file downloads, but it’s empty, missing rows, or shows numbers that don’t match what you see on screen. Almost always one of three causes: a filter you didn’t expect, the date range, or a permission scope that limits what your account can see.

Compare the on-screen view to the export

Before assuming the export is broken, verify what’s on screen before you click export:

  1. Look at the row count or totals shown in the page footer / summary area.
  2. Confirm the period selector at the top is set to the period you expected.
  3. Check every filter or drop-down on the page — it’s easy to leave a filter set from a previous session that limits the results.
  4. If row counts on screen match what the export contains, the export is doing its job — the question is why the on-screen view is narrower than expected.

If the on-screen view also looks wrong

  • Reset filters — click any “reset” or “clear” control, or refresh the page to return to default filter state.
  • Verify your permission scope — your RR Administrator controls which companies and business units you can see. If you expect to see data for a company you don’t have access to, contact your Administrator. See also the newly-granted-companies scenario if access was just granted.
  • Confirm the data should be there — if the report is filtered by “has variance” or similar, items that reconcile cleanly won’t appear.

If figures don’t match what JDE shows

RapidReconciler refreshes from JDE through a scheduled data import. If the timing of your export is between JDE updates, you may see slightly older figures. Check the “last sync” timestamp on the page if visible, and see the stale-data scenario if you suspect the refresh job has stopped.

!

If filters and permissions check out and figures still don’t match, see Email RR Support with both the on-screen view and the export attached.

Reports are slow

Reports take much longer than they used to, or the page hangs for several minutes while building results.

Quick checks

  1. Narrow the scope. Date range, business unit, or company filter — smaller scope = faster report. Confirm the report still completes when you constrain it; if so, the slowness is data volume, not a bug.
  2. Check the System Status light. A red status indicator can leave reports hanging while underlying jobs are stuck. See the system-status-red scenario.
  3. Try off-peak. The biggest reports are often run at month-end by everyone simultaneously — load on the back end can stretch run times. If you can wait, retry an hour later.
  4. Confirm it’s slow for everyone — ask a coworker to run the same report. If it’s only slow for you, see the “Blank screen” flow in Login & Access Troubleshooting for browser / network checks.
!

If reports are reproducibly slow for everyone at off-peak times and the System Status is green, see Email RR Support.

Email RR Support

If the steps in the topics above didn’t resolve your issue, email rrsupport@getgsi.com. The more context you include, the faster we can help.

What to include

  • The report or export name and where in the UI you triggered it.
  • The filters and period selector values you had set when the issue happened.
  • What you’re seeing — an error message, a hang, an empty file, etc. Screenshots help.
  • Whether other users at your site are affected, or if it’s just you.
  • What you tried from the topics above.
  • The export file itself, if it downloaded but the contents look wrong.
