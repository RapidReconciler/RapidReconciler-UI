Overview

A Zero Balance Adjustment is automatically created in the General Ledger when the inventory quantity for an item goes to zero while a cost value remains associated with that item. This occurs regardless of which inventory cost method is selected in the Cost Revisions file (F4105).

The underlying principle is that an inventory value cannot exist if there is no quantity on hand. When this condition is detected, the system automatically generates a journal entry to bring the inventory value to exactly zero.

Zero Balance Adjustments apply to the Purchasing and Inventory modules only. The Sales system was not designed to use this functionality.

Section 1: Why Zero Balance Adjustments Occur

A Zero Balance Adjustment is triggered when two conditions are both true after a transaction is processed:

  1. The inventory quantity on hand reaches zero (or crosses through zero).
  2. A positive or negative cost value still remains associated with the item after the transaction.

The most common cause is a cost difference — the cost at which inventory was removed differs from the cost stored in the F4105 Cost Revisions file prior to the transaction. This cost difference leaves a residual dollar value that cannot remain in the inventory account with no quantity to support it.

Section 2: Purchasing Zero Balance Adjustments

2.1DMAAI Configuration

Zero Balance Adjustments in the Purchasing module are controlled by the following AAIs in the Distribution/Manufacturing AAI Values table (F4095):

AAIAccountDirectionDocument Type
4400InventoryDebitIB (default)
4405COGS or ExpenseCreditIB (default)

These AAIs must be configured for the correct company, document type, and GL class code combination. If AAI 4400 or 4405 is missing or points to an incorrect account, the zero balance adjustment will either fail or post to the wrong GL account, creating a reconciliation variance.

For full AAI configuration guidance, see the DMAAI Reference Guide.

2.2Program

Zero Balance Adjustments in Purchasing are generated by:

  • P4312 — Enter Receipts by Purchase Order or by Item

Section 3: Inventory Zero Balance Adjustments

3.1DMAAI Configuration

Zero Balance Adjustments in the Inventory module are controlled by the following AAIs in F4095:

AAIAccountDirectionDocument Type
4126InventoryCreditIB (default)
4128COGS or ExpenseDebitIB (default)

As with Purchasing, these AAIs must be correctly configured for the applicable company, document type, and GL class code. Missing or incorrectly mapped AAIs will result in posting errors or reconciliation variances.

For full AAI configuration guidance, see the DMAAI Reference Guide.

3.2Programs

Zero Balance Adjustments in Inventory are generated by:

  • P4112 — Inventory Issues
  • P4113 — Inventory Transfers
  • P4114 — Inventory Adjustments

Section 4: Sales Module

The Sales system was not designed to use Zero Balance Adjustments. Sales transactions are processed through Sales AAIs 4210 and 4220 (inventory and COGS), and the zero balance adjustment AAIs (4126/4128, 4400/4405) are not invoked by any Sales program.

If a zero balance adjustment appears to be needed in the context of a sales transaction, the root cause should be investigated in the Inventory or Purchasing modules where the on-hand quantity was last affected.

Section 5: Identifying Zero Balance Adjustments in the Item Ledger

Zero Balance Adjustments appear in the Item Ledger (F4111) and can be identified by the following characteristics:

FieldValue
Document TypeIB
Explanation field"Zero Balance Adjustment"
QuantityZero (the adjustment affects value only, not quantity)
Extended CostThe residual dollar amount being cleared

When reviewing the cardex in Excel, zero balance adjustments should be included in the extended cost summary but excluded from quantity totals (since their quantity is zero). They do not carry a posting code of "X" and are not memo transactions — they represent actual GL postings.

Section 6: Reconciliation Considerations

Zero Balance Adjustments post to AAIs 4126/4128 or 4400/4405 depending on the module. If these AAIs are configured to point to non-inventory accounts (such as COGS or expense accounts), the adjustments will not appear in the inventory GL account balance, potentially creating a discrepancy between the Stock Status report and the Trial Balance.

Best Practice: Configure AAIs 4400 and 4126 to point to the same inventory account used for standard inventory transactions (AAIs 4310 and 4122 respectively). This ensures that zero balance adjustments remain within the inventory account and do not create unexplained variances.

If zero balance adjustments are posting to expense or COGS accounts and creating reconciliation variances, review the AAI configuration for 4405 and 4128 and consider whether the offset account is appropriate for your organization's financial reporting requirements.

For guidance on the broader reconciliation context, see the Stock Status and Trial Balance Reconciliation guide.
