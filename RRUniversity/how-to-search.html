How-to guide

How to find the right answer fast.

RapidReconciler University has four search behaviors most people don’t discover on their own. Knowing them takes about three minutes and saves a lot of scrolling.

Type into the search box at the top of the University home page and results filter live. Two things shape what comes back: how you phrase your search and which module pills you have on. This page walks through both.

Part 1Two ways to ask

The search treats keywords and questions as two different modes. Each returns a different set of results, and a coach line below the search box tells you which mode you’re in as you type.

Keyword mode

Tight list of matching docs

Type two or three keywords. Returns only documents whose title or section heading contains every term. Best when you know what you’re looking for and want a short list.

gl class code

Result: ~8 matches across 2 docs. Top hit: GL Class Code Management & Change Procedures.

period close

Result: ~5 matches in Period Close Troubleshooting, Start Here Inventory, and a few section anchors. Every match has “period close” in its title or heading.

Question mode

Full content of every doc

Phrase as a question (or end with ?). Returns matches from doc body text, not just titles. Best when you don’t know which doc covers your topic, or want every mention.

how are gl class codes used

Result: ~20+ matches across 10+ docs. Includes Cardex, Transfer Order Reference, and Purchase Order Reference because each mentions GL class codes in passing.

period close?

Result: Wider set than keyword mode. The trailing ? alone is enough to switch modes — no question word needed.

Both ? at the end and natural question phrasing (“how do I…”, “what is…”, “why does…”) trigger question mode. Trailing punctuation like . or ! is neutral — ignored either way.

Tip. Start with keyword mode — it almost always returns the most relevant doc as the first result. If nothing useful surfaces, rephrase as a question to widen the search across full doc text.

Part 2Tell University you have a problem

By default, question mode hides Troubleshooting docs from your results. Most questions are instructional (“how do I…”), so University assumes you want walkthroughs, not error catalogs.

When you actually do have a problem, you opt back into Troubleshooting by including a trouble word or a declarative trouble statement:

If your query contains…Behavior
error · fail · fix · stuck · broken · wrongTroubleshooting docs included alongside instructional
can’t · isn’t · won’t · doesn’t · not workingSame — contractions count as trouble words
My period close failed (no question phrasing)Declarative trouble statement — same treatment
No trouble words, just a how-do-I questionTroubleshooting docs suppressed — you see only instructional walkthroughs

The coach line tells you what’s going on. After typing how do I close a period you’ll see:

Coach: Showing instructional docs. Add error, fail, or fix if you also want Troubleshooting.

After typing my period close failed:

Coach: Troubleshooting docs included — your query mentions a problem.
Quick rule. If you’re learning how to do something, ask a clean question. If something’s broken, mention the trouble word explicitly or just describe the symptom (“system status light is red”, “can’t log in”) — the matching runbooks surface.

Part 3Filter to your module

The four pills under the hero — Inventory A/P Transfers Administrators — let you scope the search to the modules you actually use.

  • Default: only Inventory is on. Toggle the others as needed.
  • Persistent: your selection survives reloads (stored locally in your browser).
  • General docs always appear: the four Start Here docs, the UI Reference, and Getting Started with RapidReconciler show up regardless of which pills are on.

The pills also drive the Onboarding & Workflow drawer and the Browse all documents drawer. Toggle a pill on and the corresponding role card and doc group appear inside each drawer; toggle off and they hide. The drawers always reflect your current pill state.

Don’t forget to toggle. If you’re working in A/P or Transfers and only Inventory is on, those docs won’t appear in your search results. Flip the right pill on first.

Part 4Quick reference

Cheat sheet of every search behavior on one screen. Keep this handy until the patterns are second nature.

You type…You get…

a question (or anything ending in ?)Question mode — full body search, wider result list

nonsense (xyzqwertynonsense)Coach line: No matches. Try fewer terms, check spelling, or rephrase as a question.

The coach line as you type

The italic line below the search box is the most direct teacher. It updates after every keystroke (with a 100 ms debounce) and tells you which mode is active and how to widen or narrow:

  • Empty: intro about keyword vs question modes
  • Keyword + titles matched: “Showing title matches only. Phrase as a question to widen.”
  • Keyword + body fallback: “Phrase as a question to search across the full doc text.”
  • Question, no trouble: “Showing instructional docs. Add error, fail, or fix to include Troubleshooting.”
  • Question with trouble: “Troubleshooting docs included — your query mentions a problem.”
  • Zero matches: “No matches. Try fewer terms, check spelling, or rephrase.”

The coach is the fastest way to learn the search — just watch it as you type and the system tells you what it’s doing.

One last thing. If your search isn’t returning what you expect, the most common fix is to flip a module pill on. The second-most-common fix is to phrase your query as a question. The third is to add a trouble word. In that order.
