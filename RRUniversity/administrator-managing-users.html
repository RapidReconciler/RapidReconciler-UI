User Management

Add, edit, and remove RapidReconciler users. Configure their authorized functions, visible tabs, and which companies they may access in each module.

Navigation

User management is accessed by navigating to Admin > Users in the main navigation panel. The Users page displays a list of all current users in the RapidReconciler application, showing their active status, full name, username, and available options.

User Actions

The following actions are available from the Users page:

ActionHow to Access
Add a new userClick the New User button in the top right corner.
Edit user detailsClick the paper/pencil icon in the center of the Options column.
Edit user rightsClick the lock icon on the left of the Options column.
Delete a userClick the trash can icon on the right of the Options column, then confirm the prompt.

Adding or Editing a User

Clicking the New User button or the paper/pencil icon opens the user details pop-up window:

Complete the following fields:

FieldDescription
Full NameThe first and last name of the user.
EmailThe company email address of the user. The user will automatically receive an email regarding the confidentiality policy upon account creation.
PasswordAssign an initial password. The minimum length is 8 characters. If complex passwords have been enabled for your company, the password must meet those requirements. A green check appears next to the field label when the entered value meets the configured requirements.
Temporary password (1 day)Set to true to require the user to reset their password after 1 day. Recommended for new users.
ActiveSet to true for active users. Setting this to false prevents the user from logging in until it is changed back.

Click Save Changes in the lower right when complete to save the user record.

Editing User Rights

Click the lock icon on the Users page to edit the rights and permissions for a specific user. The user rights window is divided into three sections, each controlling a different layer of access:

  1. Authorized Functions — what high-impact operations the user can perform
  2. Authorized Tabs — which modules the user can see in the navigation
  3. Authorized Companies — which companies the user can view within each visible module

Each layer is independent. Granting access at one layer does not automatically grant it at the others — a user who can see the Inventory tab still needs at least one company enabled in the Authorized Companies matrix to actually view inventory data.

Authorized Functions

Authorized functions control access to specific high-impact operations that go beyond the standard reconciliation workflow. All four functions are off by default — they must be explicitly granted, and most should be reserved for administrators or power users.

FunctionDescriptionRecommended For
Import JDE Grants permission to trigger an ad-hoc data import from JD Edwards outside the regular nightly cycle. Because this can affect data consistency for other users mid-period, it is rarely recommended. Only when GSI advises; not needed for normal operation.
Restart Service Grants permission to restart RapidReconciler data services from within the application. Used to recover from issues exporting inventory As-Of data to Excel and to clear memory on the application server. The restart process takes approximately 5 to 10 minutes during which the application is unavailable to all users. RapidReconciler administrators only.
In Transit Exclude Grants the ability to exclude order pairs in the In Transit module. Exclusions affect what is reported as in transit and require investigation in JD Edwards before being applied. Administrators and senior reconcilers.
PO Receipts Suspend Grants the ability to suspend purchase orders from contributing to the PO Receipts reconciliation. Has similar implications to In Transit Exclude. Administrators and senior reconcilers.
Principle of least privilege

Grant Authorized Functions only to users who need them for their job. The Restart Service permission in particular affects every other active user, so it should be tightly held.

Authorized Tabs

Authorized tabs control which modules are visible in the main navigation panel. This is the coarsest layer of access — a tab that is unchecked here will not appear in the user's navigation at all, regardless of any company-level access configured below.

TabWhat It Reveals
InventoryThe Inventory module — perpetual vs. GL reconciliation, transactions, As-Of, and integrity reports.
In TransitThe In Transit module — transfer order reconciliation, orders, transactions, and integrity reports.
PO ReceiptsThe PO Receipts module — Received-Not-Vouchered reconciliation.
AdminThe administration module — companies, users, general settings, and offset accounts. Only grant this to RapidReconciler administrators.
Common configurations

Inventory analyst: Inventory tab only.
In Transit specialist: Inventory and In Transit tabs.
Full reconciler: Inventory, In Transit, and PO Receipts tabs.
Administrator: All four tabs, including Admin.

Authorized Companies

The Authorized Companies section is a matrix that controls which companies the user can view, broken down by module. Each row is a licensed company; each column is a module the user has access to (per the Authorized Tabs section above).

This per-module breakdown means access can be tailored precisely. A reconciler responsible for Inventory across all U.S. entities but only In Transit at headquarters can be configured exactly that way — Inventory checked for every U.S. company, In Transit checked only for the headquarters company.

Behavior

  • Only companies the user has access to appear in the company filter dropdowns within the application.
  • Reconciliation totals shown to the user only include authorized companies — figures and variances on the Reconciliation page reflect only what the user is permitted to see.
  • If a module column is unchecked for every company, the user will see the module's tab (if granted in Authorized Tabs) but no data within it.
  • If Authorized Tabs does not include a particular module, that module's column in this matrix has no effect — the user cannot reach the module to use the access.
Adding new companies

When a new company is added to your RapidReconciler license, it appears in this matrix unchecked for every existing user. Each user who needs access to the new company must be updated individually.

Click Done when all rights have been configured.

Common Pitfalls

User can't see a company they should have access to

Both layers must be granted: (1) the relevant module must be checked under Authorized Tabs, and (2) the company must be checked in the matching column under Authorized Companies. If either is missing, the user won't see the data.

User sees the tab but no data inside it

The Authorized Tabs section grants visibility of the module, but the per-company matrix controls what data appears inside. Confirm at least one company is checked in that module's column.

Reconciliation totals look wrong for a user

Reconciliation figures shown to a user only include companies they are authorized for in the relevant module. If a user reports unexpected totals, verify their Authorized Companies matrix matches the scope of their job.

Deleting a User

To delete a user, click the trash can icon to the right of their name on the Users page and confirm the prompt when displayed.
