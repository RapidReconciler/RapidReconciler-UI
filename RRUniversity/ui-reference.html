UI Reference

Find any indicator, button, icon, panel, or status in RapidReconciler — what it’s called, what it tells you, and what to do when you see it. Search by name, by symptom (“red light”, “trash icon”), or browse by area.

Web Page Banner 6 elements · Top-of-page controls visible from any screen.

These six controls live in the banner at the very top of the application, to the right of the GSI logo.

Collapse button

Where you’ll see it
Top-left of the banner, just to the right of the logo. The icon shows a stack of horizontal lines.
What it does
Hides or shows the left-side navigation panel.
What you should do
  • Click it any time you want more horizontal space — wide tables on the Reconciliation page are the most common reason.
  • Click again to bring the navigation panel back. Your selection persists for the session.
Also called
hamburger menu, nav toggle, sidebar toggle, hide nav

Personal Data toggle

Where you’ll see it
Banner, right-hand side. Icon shows a person silhouette.
What it does
Toggles the display of personal user information (your full name and the system data block).
What you should do
  • Turn it off when you're sharing your screen and don't want personal info captured in screenshots or recordings.
  • Turn it back on after the session — there's no security implication, this is purely a display preference.
Also called
hide name, anonymize, screenshare mode, privacy toggle

Log Out button

Where you’ll see it
Banner, right-hand side. Icon shows a door with an arrow.
What it does
Ends the current session and logs you out.
What you should do
  • Use it whenever you're finished working, especially on a shared workstation.
  • If the application appears to hang, log out and back in before opening a support ticket — it clears most transient session issues.
Also called
sign out, exit, end session

Full Screen button

Where you’ll see it
Banner, right-hand side. Icon shows four corners pointing outward.
What it does
Expands RapidReconciler to fill your entire monitor, hiding the browser tabs and address bar.
What you should do
  • Useful for reviewing wide tables (PO Receipts Orders page, Inventory Line Detail) where every column counts.
  • Press Esc or click the icon again to exit full-screen mode.
Also called
maximize, expand, presentation mode, fullscreen

System Alerts indicator

Where you’ll see it
Banner, right-hand side. Icon is a bell.
What it tells you
Displays active system notifications — typically maintenance windows, version updates, or warnings the GSI team has posted.
States
  • No badge — No active alerts. Nothing for you to read.
  • Number badge — That many active alerts are waiting. Click to read them.
What you should do
  • Click the bell to see the current alerts.
  • Read maintenance-window alerts before starting period-close work — the application may be unavailable during the window.
  • If you see an alert about an import failure, treat the System Status light on the Module Status Bar as authoritative for whether your data is current.
Also called
notifications, bell, badge, system messages, alerts

Help Panel button

Where you’ll see it
Banner, far right. Icon is a question mark.
What it does
Opens the context-sensitive help panel for the page you're currently on.
What you should do
  • Click it on any page and the help content adjusts to that page's topic — no need to search through external docs.
  • Use it as your first stop before opening a support ticket; many questions are answered in the page-level help.
Also called
question mark, ?, help, context help, page help
Main Navigation Bar 9 elements · Left-side panel with module links and global controls.

The Main Navigation Bar is the vertical strip on the left side of the page (or hidden behind the Collapse button). It's split into a system area at the top and the module list below it.

System Data block

Where you’ll see it
Top of the Main Navigation Bar.
What it does
Shows the server name you're connected to and your full name.
What you should do
  • Glance at it when troubleshooting to confirm you're on the server you think you're on (PROD vs TEST).
  • When opening a support ticket, copy or screenshot this block — it's the fastest way to tell GSI which environment you're working in.
Also called
server name, environment, user name, who am i

Database Drop-Down

Where you’ll see it
Top of the Main Navigation Bar, just below System Data.
What it does
Lets you switch between databases you have permission to view (typically Production, PY, QA, Training, or Test).
What you should do
  • Confirm the selected database before doing reconciliation work — variances differ between environments.
  • If you don't see a database you expect to access, contact your administrator to grant database-level permission.
Also called
env switcher, db dropdown, environment selector, database selector, prod test py

Import JDE Data button

Where you’ll see it
In the Main Navigation Bar. Typically only visible to Administrators.
What it does
Launches the job that imports JDE data into RapidReconciler.
What you should do
  • Don't click it casually. The import is normally scheduled overnight; manual triggering is for after a JDE issue has been corrected and you need fresh data before the next scheduled run.
  • After launching, watch the System Status light on the Module Status Bar — it turns green when the import completes.
  • If you don't see this button, your account doesn't have admin rights. That's normal for non-admin users.
Also called
manual import, refresh data, run import, jde sync, force import

Restart Service button

Where you’ll see it
In the Main Navigation Bar. Typically only visible to Administrators.
What it does
Restarts the RapidReconciler data services on the application server.
What you should do
  • Use it when users report download issues (Excel exports failing, reports hanging) or when the System Status light goes red and stays red.
  • A restart is brief but disruptive — warn active users before clicking. The application will be unavailable for ~30 seconds.
  • If the issue persists after a restart, escalate to GSI rather than restarting repeatedly.
Also called
service restart, fix downloads, restart RR services, recycle services

Inventory module link

Where you’ll see it
Main Navigation Bar, in the modules list.
What it does
Opens the Inventory module — perpetual inventory to General Ledger reconciliation.
What you should do
  • Use it when you're reconciling inventory accounts at period close.
  • If the link is missing, your account doesn't have Inventory module rights. Contact your administrator.
Also called
inventory tab, perpetual inventory, GL reconciliation

In Transit module link

Where you’ll see it
Main Navigation Bar, in the modules list.
What it does
Opens the In Transit module — goods-in-transit (ST/OT orders) to GL reconciliation.
What you should do
  • Use it when you're reconciling transfer orders that have shipped but not yet received.
  • If the link is missing, your account doesn't have In Transit rights. Contact your administrator.
Also called
in transit tab, transfer orders, ST OT, in-transit reconciliation

PO Receipts module link

Where you’ll see it
Main Navigation Bar, in the modules list.
What it does
Opens the PO Receipts module — Received-Not-Vouchered (RNV) reconciliation.
What you should do
  • Use it when you're investigating the RNV balance, suspending stuck orders, or clearing aged receipts.
  • If the link is missing, your account doesn't have PO Receipts rights. Contact your administrator.
Also called
PO receipts tab, RNV, received not vouchered, voucher match

Admin module link

Where you’ll see it
Main Navigation Bar, in the modules list. Only visible to administrators.
What it does
Opens the Admin module — companies, users, general configuration.
What you should do
  • Use it for user management and licensed-company management.
  • If the link is missing, your account doesn't have admin rights. That's normal — non-admins should never see Admin.
Also called
admin tab, administration, user management, settings

University link

Where you’ll see it
Main Navigation Bar, bottom of the modules list. Visible to all users.
What it does
Opens RapidReconciler University — videos, walkthroughs, and reference material.
What you should do
  • This is the documentation you're reading right now.
  • Bookmark common pages (your role's Start Here guide, the relevant Walkthrough) for quick access during reconciliation work.
Also called
university link, knowledge base, docs, help, training
Module Status Bar 5 elements · The horizontal strip at the top of every module page — page identifier, two health indicators, and period selector.

The Module Status Bar shows you (a) where you are, (b) whether your data is healthy, and (c) which period you're looking at. It contains four elements left to right.

Page Identifier

Where you’ll see it
Far left of the Module Status Bar.
What it does
Displays the name of the page you're currently on (e.g., "Reconciliation", "Orders", "Line Analysis").
What you should do
  • Use it to confirm you're on the right page before acting — especially after deep-linking from a support ticket or another doc.
Also called
page name, breadcrumb, where am i

Inventory Validation Status Light

Where you’ll see it
Module Status Bar, center-left. Inventory module only — the equivalent in PO Receipts is the PO Receipt Validation light.
What it tells you
Whether the inventory roll-forward from the prior closed period to the period you're working on reconciles cleanly.
States
  • Green light — Carry-forward from the prior period is accurate.
  • Red light — Prior period issue exists — typically an unposted batch that must be resolved before making journal entries this period.
What you should do
  • Hover the light to see the detail message — it tells you what specifically is wrong.
  • If green, continue with this period's reconciliation — no action needed.
  • If red, stop. Don't continue this period's reconciliation until the prior-period variance is resolved. Open the prior closed period, find the source of the variance, and clear it. See the Inventory Reconciliation Walkthrough — Step 1 for the full diagnostic procedure.
  • If you don't have access to the prior period (closed/locked), have your administrator reopen it for diagnosis.
Also called
red light, green light, validation indicator, prior period indicator, top center light, roll forward indicator

PO Receipt Validation Status Light

Where you’ll see it
Module Status Bar, center-left. PO Receipts module only.
What it tells you
Whether the PO receipts carry-forward from the prior closed period to the period you're working on reconciles cleanly.
States
  • Green light — Carry-forward from the prior period is accurate.
  • Red light — Prior period issue exists. Resolve it before proceeding with this period's reconciliation.
What you should do
  • Hover the light to see the detail message.
  • If red, do not continue this period's RNV reconciliation. Resolve the prior-period variance first. See the PO Receipts Reconciliation Walkthrough for the procedure.
Also called
red light, green light, PO validation, RNV validation, prior period light

System Status Light

Where you’ll see it
Module Status Bar, center-right. Visible across all modules.
What it tells you
Whether the JD Edwards data import into RapidReconciler completed successfully overnight.
States
  • Green light — JDE import completed successfully — your data is current.
  • Red light — Import error occurred. Your data may be stale or incomplete.
What you should do
  • Hover the light to see the detail message.
  • If green, continue — your data reflects the latest JDE state through the last import window.
  • If red, do not trust today's reconciliation numbers as authoritative. Contact your RR Administrator to investigate the import failure. Common causes: JDE-side database changes, network outages between RR and JDE, expired credentials.
  • After the administrator triggers a successful re-import (via the Import JDE Data button), the light should turn green within a few minutes. Refresh the page to confirm.
Also called
red light, green light, import status, JDE import, system health, data freshness, stale data, import failed

Period End Selector

Where you’ll see it
Far right of the Module Status Bar.
What it does
Lets you choose which fiscal period you're reconciling. Available on modules that need it.
What you should do
  • Always pick the most recently closed period, not the current open one — reconciliation is a backward-looking exercise.
  • If you don't see the period you want, scroll the dropdown — older periods may be below the visible range.
  • Changing the period reloads the page data; if numbers look wrong after a change, give the page a moment to refresh before investigating.
Also called
period selector, period end, period dropdown, fiscal period, change period, last period
Inventory Module 4 elements · Reconciliation page elements specific to perpetual-inventory-to-GL reconciliation.

These elements live on the Inventory module's Reconciliation page. The status lights and period selector at the top are documented separately under Module Status Bar.

Account Filter

Where you’ll see it
Left side of the Reconciliation page, four columns.
What it does
Filters the displayed reconciliation to a specific account, organized by JD Edwards account hierarchy: Company → Business Unit → Object → Subsidiary.
What you should do
  • Pick from left to right — selecting a Company narrows the Business Unit list, and so on.
  • You can stop at any level: leaving Subsidiary blank includes all subsidiaries of the selected Object account.
  • If your filter selection produces no rows, double-check the period selector — the account may have no activity in that period.
Also called
account filter, four columns, company business unit object subsidiary, JDE account picker

Valuation Section

Where you’ll see it
Reconciliation page, after the Account Filter.
What it tells you
Three numbers that frame the entire reconciliation: GL Balance, Perpetual Balance, and Out of Balance (the difference).
States
  • GL Balance — The dollar value in the General Ledger for the filtered account at period end.
  • Perpetual Balance — The dollar value calculated from the perpetual inventory records for the same account at the same period.
  • Out of Balance — The variance between the two. Zero (or near zero) means the account is reconciled.
What you should do
  • A non-zero Out of Balance is your starting point — it's the dollar amount you need to explain via the Variance Calculation section below.
  • The GL Balance figure should match what you see in JDE for the same account and period. If it doesn't, something is wrong with the import (check the System Status light).
Also called
GL balance, perpetual balance, out of balance, variance, valuation summary, headline numbers

Variance Calculation Section

Where you’ll see it
Reconciliation page, below the Valuation Section.
What it does
Six lines, one per known variance source. Together they decompose the Out of Balance figure into its causes.
What you should do
  • Focus on the non-zero lines — those are where the variance is actually coming from.
  • Click into each non-zero line to drill down to the documents driving it.
  • A small residual after explaining the major lines is normal (rounding, immaterial timing). A large unexplained residual means a variance source not yet captured by the calculation — escalate to GSI.
Also called
variance breakdown, six lines, variance sources, drill down, decomposition

Out of Balance History Graph

Where you’ll see it
Reconciliation page, lower section.
What it does
A line chart showing the Out of Balance figure across the last 14 closed periods.
What you should do
  • Use it as a sanity check — is this period's variance an anomaly or part of a trend?
  • A growing variance over time means a structural problem (DMAAI misconfiguration, unposted batches accumulating). Address the root cause; don't just clear this period.
  • A spike in a single period usually means a one-time event (manual journal entry, reversed receipt out of sequence). Investigate that period specifically.
Also called
history chart, trend chart, 14 periods, variance trend, balance over time
PO Receipts Module 6 elements · Elements specific to Received-Not-Vouchered (RNV) reconciliation.

These elements live on the PO Receipts module's Orders page, Reconciliation page, and Line Analysis page. The status lights at the top are documented under Module Status Bar.

Calculation Section

Where you’ll see it
PO Receipts Reconciliation page, prominent block at the top.
What it does
Decomposes the total RNV out-of-balance figure into its components: Open Receipts, RNV Tax, Landed Cost, and others.
What you should do
  • Read it top-down — Open Receipts is usually the largest component.
  • A non-zero Tax line typically means tax was added at voucher but not at receipt (or vice versa). Compare F43121 tax fields to the F0911 entries to confirm.
  • A non-zero Landed Cost line with no expected landed-cost activity often means a PRLAND = 3 record. Suspend those rather than chasing them.
Also called
calculation breakdown, RNV components, open receipts, RNV tax, landed cost

PO Receipts Aging Chart

Where you’ll see it
PO Receipts Reconciliation page.
What it does
Visual breakdown of open receipts by age (current, 30, 60, 90+ days).
What you should do
  • Healthy: most balance sits in the 0–60 day buckets — those are receipts waiting on normal voucher matching.
  • Unhealthy: significant balance in the 90+ bucket. Those are aged open receipts that need investigation: late supplier invoicing, PRLAND = 3 records, or PO's closed in JDE without proper voucher match.
  • Click a bar to filter the order list to that age band, then work the items individually.
Also called
aging chart, aging breakdown, 90 days, aged receipts, old receipts, aging buckets

Reconciled / Suspended Filters

Where you’ll see it
Top of the PO Receipts Orders page.
What it does
Toggle filters that let you focus on unreconciled orders by hiding orders that have already been suspended or reconciled.
What you should do
  • Default view (both filters off) shows everything — useful for full-list audits.
  • Turn ON "Hide Suspended" to focus on orders still requiring action.
  • Turn ON "Hide Reconciled" to drop already-cleared orders.
  • For routine period-close work, both ON gives you the actionable working list.
Also called
orders filter, hide suspended, hide reconciled, working list, focus filter

Edit Note button

Where you’ll see it
Line Analysis page, action area.
What it does
Single point of control for adding, editing, or removing a note on a suspended or excluded item.
What you should do
  • Always add a note when you suspend an order — future you (or the next reconciler) needs to know why.
  • Notes are searchable in the Orders page. A consistent note vocabulary ("PRLAND=3", "manual voucher", "aged - awaiting credit") makes audit reviews much faster.
  • Editing or removing a note doesn't un-suspend an order — that's a separate action via the Suspended checkbox.
Also called
note button, edit note, add note, suspension note, comment

Document button

Where you’ll see it
Line Analysis page, action area.
What it does
Opens a side-by-side comparison of the F43121 (PO receiver) record against the F0911 (GL) record for the same document.
What you should do
  • Use it when a specific PO has a small variance you can't explain — it usually exposes penny-rounding, tax-explanation-code differences, or out-of-sequence postings.
  • If the F43121 and F0911 amounts match exactly but the order still shows variance, the issue is on a different line of the same document. Check sibling lines.
Also called
document button, F43121 F0911 comparison, side by side, line compare

Suspension Feature

Where you’ll see it
Available on every order in the Orders page (Suspended checkbox).
What it does
Removes an order from the active reconciliation total without modifying the underlying JD Edwards data.
What you should do
  • Suspend orders that genuinely don't belong in the active RNV picture: PRLAND = 3 landed costs, PO's at status 999 with no further activity possible, manual GL clearances.
  • Suspension is reversible — un-suspending puts the order back in the active total. Use this if new activity posts to a previously-suspended order.
  • Always pair a suspension with a note (Edit Note button) explaining why. See the Suspension & Exclusion guide for the full decision tree.
Also called
suspend, suspension, remove from total, exclusion, suspended orders
Admin Module 5 elements · User management and configuration UI. Visible only to administrators.

These elements appear in the Admin module. If you don't have admin rights, you'll never see them — that's by design.

New User button

Where you’ll see it
Top right corner of the Users page (Admin module).
What it does
Opens the New User Detail panel where you create a user record.
What you should do
  • Click it when you need to add a new user.
  • Have the user's name, email address, and intended rights (which modules, which databases) ready before you click — partial records can't be saved.
  • If the button is missing or grayed out, your account doesn't have user-management rights. Contact your RR Administrator.
Also called
add user, create user, +User, new user

Edit user details icon

Where you’ll see it
Center of the Options column on the Users page. Icon shows a paper with a pencil.
What it does
Opens the User Detail panel for editing the selected user's name, email, and basic info.
What you should do
  • Use it for routine updates — name change after marriage, email address change after a domain migration.
  • Edits are immediate and live — there's no draft or approval step.
Also called
paper pencil icon, edit details, edit user, modify user, pencil icon

Edit user rights icon

Where you’ll see it
Left of the Options column on the Users page. Icon is a padlock.
What it does
Opens the user-rights editor where you grant or revoke access to specific modules and databases.
What you should do
  • Use it when a user changes roles or needs access to a new module.
  • Rights changes take effect on the user's next login — they may need to log out and back in to see the change.
Also called
lock icon, padlock, user rights, user permissions, access, grant access

Delete user icon

Where you’ll see it
Right of the Options column on the Users page. Icon is a trash can.
What it does
Removes the user from the system.
What you should do
  • Prefer setting Active = No to outright deletion. Deactivation preserves history (suspensions, notes, audit trail) attributable to that user; deletion may remove some of that linkage.
  • Use deletion only for accounts that were created in error and have no activity attached.
  • Deletion cannot be undone — the user record is gone, not just hidden.
Also called
trash can, delete user, remove user, deactivate

Authorized Tabs

Where you’ll see it
On the user-rights editor (opened via the lock icon).
What it does
Four checkboxes that control which modules a user can access: Inventory, In Transit, PO Receipts, Admin.
What you should do
  • Grant only what each user needs. Most reconcilers need only one or two module tabs.
  • "Admin" should be granted sparingly — it's the user-management tab and any other administrative function.
  • Unchecked tabs simply don't appear in that user's Main Navigation Bar.
Also called
authorized tabs, module access, tab permissions, inventory in transit po receipts admin
