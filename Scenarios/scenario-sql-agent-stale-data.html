Stale or wrong data

Data looks old and isn’t refreshing

Figures in RapidReconciler are showing yesterday’s or older numbers and aren’t updating. RapidReconciler refreshes from JDE on a nightly cadence, so the first question is simply whether the refresh window has closed yet. If it has and the data is still stale, the scheduled refresh job on your database server has likely stopped or is failing — that fix requires your IT or DBA.

What you’re seeing

  • RapidReconciler is showing yesterday’s or older figures rather than current data.
  • You expect a daily (or scheduled) refresh that hasn’t happened.
  • Other parts of RapidReconciler load and look healthy — only the data is old.

Try these steps in order

Step 1
Confirm the nightly refresh window has closed and the data really is stale

RapidReconciler imports from JDE on a nightly cadence. If you’re looking before the refresh has run for the day, the figures will reflect yesterday’s state — that’s expected, not a problem. Verify the refresh is overdue before assuming something’s broken.

  1. Check the “last updated” or “last sync” timestamp on the page (or hover the status indicators — the tooltip mentions the refresh time).
  2. Compare against your customer’s expected refresh window. If the timestamp is from after last night’s window, today’s data is in.
  3. Hard-reload the page (Shift+F5 on Windows, Cmd+Shift+R on Mac) to make sure you’re not looking at a cached older version.
  4. Confirm the figures match what your ERP showed at the same point yesterday (or the prior expected refresh time).
Refresh overdue and figures genuinely stale? Continue to Step 2. Otherwise, stop here — this is normal pre-refresh behavior.
✉
Hand off to your IT
The remaining checks need to be performed by your IT department or DBA. Click to draft an email with the steps below pre-filled.
Step 2 For your IT
Ask your IT or DBA to check the SQL Agent job

The refresh runs through a SQL Agent job on your database server. If that’s stopped, disabled, or failing, data won’t flow. Forward this to your IT or DBA:

  • Confirm the SQL Server Agent service is running on the database server (and set to start automatically).
  • In SSMS, expand SQL Server Agent → Jobs and confirm the RapidReconciler job is enabled.
  • Right-click the job → View History. Check the last several runs — succeeding, failing, or not running at all?
  • If a recent run failed, expand it and read the step-level error message.
  • If the job is healthy but data is still stale, run it manually (Start Job at Step) and watch it complete.
Resolved? Stop here. Otherwise, continue below to email RR support.

Still not resolved?

Email RapidReconciler support

If your IT/DBA confirms the SQL Agent job is healthy but data is still stale, email rrsupport@getgsi.com with:

  • The timestamp the data is stuck at, and what timestamp you expected to see.
  • What your IT/DBA found when checking the SQL Agent job (job state, last run result, any error messages).
  • A screenshot showing the stale figures with the company / business unit visible.
