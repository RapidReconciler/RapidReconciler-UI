Agent install hangs at "Validating Data" — database connectivity blocked at install time.

During agent install, the browser displays "Validating Data" indefinitely instead of progressing to the SQL Server connection prompt or "Deploying". The agent can't reach the database server — either DNS, firewall, credentials, or the database itself is the blocker. Verify each connection layer in turn before escalating.

Section 2

Possible solutions — ordered roughly easiest to hardest.

1 Verify database connectivity from the application server, then escalate to DBA Path 4 · Escalate
When this applies: Agent install reaches "Validating Data" but never advances. The most common root cause family is the application server can't reach the database server — bad SQL port, blocked firewall, missing database, or invalid credentials.
Steps to resolve
  1. From the application server, attempt a manual connection to the database server on the configured port to verify reachability.
  2. Confirm the database, the rruser login, and SQL Authentication mode are all in place per the pre-install database setup.
  3. If the prompt is reached but still fails, double-check the values being entered match the customer's SQL configuration exactly.
  4. Check Windows Event Viewer on the application server for RapidReconciler-related errors.
  5. Check today's agent .out and .err log files (see Diagnostic Tools).
  6. If the install needs to be restarted, ensure Setup Step is set back to "Created" in VALC before retrying.
