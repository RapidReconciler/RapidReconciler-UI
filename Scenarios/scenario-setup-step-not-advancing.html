Setup Step not advancing automatically after install — Setup Step field stuck at "Created" or earlier.

After completing an install or migration, the Setup Step field on the Client Details tab remains at "Created" (or an earlier value) and does not advance. Either the install didn't finish cleanly, or the Setup Step legitimately needs to be moved manually for this install path.

After completing an install or migration, the Setup Step field on the Client Details tab remains at "Created" (or an earlier value) and does not advance.

Section 2

Possible solutions — ordered roughly easiest to hardest.

1 Verify install completed cleanly, then advance Setup Step manually if needed Path 3 · Research / docs
When this applies: Setup Step is not advancing automatically. First, confirm the install genuinely finished (agent + database connectivity + services). If everything checks out, the state may need to be moved manually depending on the install path.
Steps to resolve
  1. Confirm the install actually completed end-to-end: agent installed, database showing Online, connectivity row showing Connected.
  2. If all of the above are good, manually advance the Setup Step on the Client Details tab to the appropriate next value.
  3. If you're not sure what value to set, check another active client for reference.
  4. Save the change and refresh the Clients grid to confirm the row updates.
