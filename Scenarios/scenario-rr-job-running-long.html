Job won’t finish

RapidReconciler job running longer than usual

You log in at your usual time and the System Status light is flashing amber instead of solid green. Hover over the light — the pop-up shows how long the current run has been going and the average time it normally takes. If the current run is considerably longer than the average, the job is stuck on something on the SQL Server side and your IT needs to step in.

Different symptom: if the status light is red (not flashing amber), the job has clearly failed — see System status light is red instead.

What you’re seeing

  • The System Status light is flashing amber when you log in at your usual time — you’d normally expect solid green.
  • Hovering over the light shows a pop-up with the current run duration and the average run time. The current run is considerably longer than the average.
  • Common phrasings: “RR is running long,” “the job didn’t finish,” “data isn’t updating,” “the light’s been amber all morning.”

Try this first

Step 1
Confirm the run is considerably longer than the average, then contact your IT

Hover over the System Status light and read the pop-up:

  • Current run duration — how long today’s job has been running.
  • Average run time — how long it usually takes to complete.

If the current run is roughly in line with the average, give it a little more time and check back. If it’s considerably longer than the average, the job is stuck on the SQL Server side — draft a note to your IT using the button below. The email asks them to stop the SQL job, let it roll back, and re-run it (the standard fix), and includes the deeper checks if it hangs again.
