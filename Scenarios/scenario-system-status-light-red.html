System status light is red

You log in and the System Status light in the page header is red instead of green. That means last night’s RapidReconciler job didn’t complete cleanly, so today’s figures aren’t reliable — don’t run your month-end close until the light is green again. Send a screenshot to your IT using the button below; the email pre-fills the checks they’ll need.

Different symptom: if the light is flashing amber rather than red (the job is still running, not failed), see RapidReconciler job running longer than usual.

What you’re seeing

  • The System Status indicator in the page header has turned red. The colour is the same on every module — it reflects the SQL Agent job, not the page you’re on.
  • Common phrasings: “system status is red,” “the red light is on,” “the job failed,” or “something crashed.”
  • Reports or scheduled processing didn’t complete the way they normally do.

Try this first

Step 1
Take a screenshot and contact your IT

Take a screenshot of the page header showing the red System Status light. The light is the same on every module, so any page works.

Then draft an email to your IT using the button below. The email pre-fills the SQL Server / SSIS checks they’ll need to run — disk and TempDB, SQL Agent Job History, and the SSIS execution report.

✉
Send to your IT
Draft an email to IT with the SQL Server / SSIS investigation checklist pre-filled. Drag your screenshot in before sending.
