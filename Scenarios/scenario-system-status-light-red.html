Status lights red For your IT

System status light is red

The System Status indicator at the top of the Reconciliation page is red or amber (anything other than green). A back-end job didn’t complete cleanly — the figures aren’t ready to reconcile yet. Don’t start your month-end close until both status lights are green; the figures you’d be working from can’t be trusted while either is off-color.

Most red lights are caused by something in the SQL Server / SSIS environment your IT owns — out of disk, a full TempDB log, a failed SQL Agent job, or an SSIS package error. Have your IT walk the checks below before involving RR support; many issues clear once the underlying environment is healthy and the job re-runs.

Different symptom: if the nightly job is still running rather than failed (no error, just hasn’t finished after many hours), see RR nightly job running long — that path covers stopping the hung SQL job and re-running it.

What you’re seeing

  • The System Status indicator at the top of the Reconciliation page has turned red.
  • Common phrasings: “system status is red,” “the red light is on,” “the job failed,” or “something crashed.”
  • Reports or scheduled processing didn’t complete the way they normally do.

Recommendations to share with your IT

Step 1
Capture the basic context

Note the following before forwarding to your IT — the same context becomes the email if it later needs to go to RR support.

  • The date and time the indicator turned red.
  • The company or business unit you were viewing.
  • A screenshot of the Reconciliation page header showing the red System Status indicator.
Captured? Use the button below to forward Steps 2–4 to your IT.
✉
Hand off to your IT
The remaining checks need to be performed by your IT department or DBA. Click to draft an email with the steps below pre-filled.
Step 2 For your IT
Check disk space and TempDB on the database server

The most common cause is the SQL Server host running out of room. Have IT check:

  • Free disk space on every drive that hosts the RapidReconciler database, the SQL Server data and log directories, and TempDB. Low or zero free space stalls the nightly job.
  • TempDB log file size and autogrowth. A full TempDB log will fail any large operation. If it’s near full, free space (shrink, restart the SQL service during a maintenance window, or expand the drive) before re-running.
  • Database log file (RapidReconciler_Prod_log.ldf) — same check, same fix path.
Disk and log space OK? Continue to Step 3.
Step 3 For your IT
Review SQL Agent Job History for the failed job

Have IT open SQL Server Management Studio → SQL Server Agent → Job Activity Monitor (or the Job History node) on the RapidReconciler database server.

  • Find the most recent failed RapidReconciler job that lines up with the red light’s timestamp.
  • Open the job’s history and read the error message on the failed step. Common patterns: out of disk, login failure, deadlock, package validation error.
  • Capture the error text — that’s the most useful artifact if this needs to escalate to RR support.
Found the failed job and captured the error? Continue to Step 4.
Step 4 For your IT
Pull the Integration Services (SSIS) report

If the failed job was an SSIS package, the SSIS catalog has a more detailed execution report than the SQL Agent log alone.

  • In SSMS, expand Integration Services Catalogs → SSISDB → (your folder).
  • Right-click the project or package → Reports → Standard Reports → All Executions.
  • Find the failed execution at the timestamp matching the red light. Open View Messages on that execution.
  • Capture the error messages — especially anything labeled Error or OnError.

If IT identifies and fixes the underlying issue (cleared disk, expanded log, fixed permissions), have them re-run the job. The status light should clear once the next successful run completes.

Issue identified and resolved by IT? Stop here — you’re done. Issue still unclear or not resolved? Continue below to RR support.

If your IT can’t resolve it — email RapidReconciler support

Send the crash details and what your IT found

If the SQL Agent / SSIS errors don’t point at an environment cause IT can fix, the click-and-export path captures the RR-side picture for support. Email rrsupport@getgsi.com and include:

  • Click the red System Status indicator on the Reconciliation page header. It exports an Excel with the crash details — the most useful attachment for RR support.
  • The SQL Agent error message and SSIS execution messages your IT captured in Steps 3 and 4.
  • What your IT already checked and ruled out (disk, TempDB, log space).
  • Your screenshot of the Reconciliation page header with the red indicator visible.
  • The date, time, and company / business unit from Step 1.
  • If clicking the indicator doesn’t produce an Excel, say so and include a screenshot of any browser console errors instead.
