Tech Roles

GSI Tech owns the customer install, the underlying RapidReconciler product, and ongoing support. Five distinct roles — this page covers what each one does, where they appear in the install workflow, and how work hands off between them.

Tech Client Management → Help Desk → Certificate Management →

The roles

Tap a card for the role’s start-here doc, or use Workflow to jump into the procedure.

RR DBA

Tech side

Owns the database and SSIS install end-to-end. Drives the 2-hour install meeting from intake through the completion email that triggers VALC.

 View details → Workflow →

RR Network Tech

Tech side

Picks up after the database is configured. Owns the entire VALC side of the install, the GSI network/Cloudflare work that supports it, and most live-customer change requests post-go-live.

 View details → Workflow →

RR DB Developer

Tech side

RapidReconciler product developer. Owns schema and code changes that are pushed across the customer base. Not involved in individual customer installs.

 View details → Workflow →

RR UI Developer

Tech side

Owns the backend configuration for the RapidReconciler web interface. Surfaces in the workflow during certificate renewal — when the wildcard cert is renewed, the backend has to be updated before per-customer agent rollout can begin.

 View details → Workflow →

Helpdesk Tech

Tech side

Monitors the rrsupport@getgsi.com mailbox, triages and resolves first-line support issues, and routes inbound work to the right tech-team member at the right time.

 View details → Workflow →
