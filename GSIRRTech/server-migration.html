Server Migration

Steps required to migrate the RapidReconciler environment to a new server. There are two migration scenarios depending on the customer’s server architecture — both end with the same VALC and Cloudflare cutover.

Audience: GSI Tech Team / DBA Maintenance window: Coordinate with client — data is unavailable during migration Press Ctrl+F (or ⌘+F) to search this page

Two Migration Scenarios

The architecture at the customer site determines which procedure to follow:

ScenarioWhat it meansProcedure
Dedicated Server The application (Agent) and the database are hosted on the same server. Dedicated Server Migration
Separate Servers The Agent and the database are hosted on different servers. Separate Servers Migration

Both scenarios end with the same final cutover steps in VALC & Cloudflare, and the Post-Migration Checklist works for either path.

Topics

TopicWhat it covers
Dedicated Server Migration Software prerequisites and a 5-step process for migrations where the Agent and database share one server.
Separate Servers Migration Application server and database server procedures for migrations split across two machines.
VALC & Cloudflare The shared cutover work that finalizes both scenarios: VALC IP updates, Cloudflare DNS, agent install, schedule enable, and validation.

Notify the client before you begin

Before beginning any migration, notify the client of the planned maintenance window. Data will be unavailable during the migration process — coordinate timing to minimize disruption.

Key principle

Don’t enable the SQL Agent job schedule on the new server until VALC and Cloudflare are updated and you’ve confirmed data is appearing in the UI. Premature enable runs the job against an unfinished cutover and can create confusing partial states.

Dedicated Server Migration

Use this procedure when the customer’s RapidReconciler Agent and database share a single server. The work falls into two parts: confirming software prerequisites on the new server, and then a 5-step migration process you run start-to-finish during the maintenance window.

Software Prerequisites

Confirm all of the following are in place on the new server before beginning the migration steps:

  • New server prerequisites
  • SQL Server Standard Edition (minimum), Version 2017 or later
  • Administrator access to complete all required tasks
  • Integration Services installed
  • Integration Services Catalog (SSISDB) created
  • RapidReconciler folder added to SSISDB
  • RRUSER account added or migrated with appropriate permissions
  • Visual Studio Community Edition with the SQL Server Integration Services extension
  • Applicable JDE database drivers (AS/400 or Oracle)
See Installing the Production Database for detailed instructions on creating the SSISDB catalog, the RapidReconciler folder, and the RRUSER account.

Migration Process

Run the five steps below in order. Each step is independent enough that you can pause between them, but the migration isn’t complete until the cutover work in VALC & Cloudflare is finished.

1
Step 1 of 5
Database Backup and Restore
  1. Back up the current production RapidReconciler_Prod database on the old server and copy the backup file to the new server.
  2. Restore the RapidReconciler_Prod database on the new server.
  3. Add the RRUSER SQL Authentication login on the new server using the script from the old server.
Verify before continuing

Run a quick row count on key tables such as F4111 on the restored database and compare against the old server. Confirming counts match before continuing saves significant rework if the restore failed silently.

2
Step 2 of 5
SSIS Package Migration

Complete steps 1–8 of the SSIS deployment procedure on the new server — copy the package, update connection strings, and deploy to the SSISDB catalog.

See Installing the Production Database → Tour 3: Configure & Deploy the SSIS Package for the detailed package-deployment steps.
Update connection strings

If connection strings reference the old server by name or IP, update them to point to the new server before deploying the package.

3
Step 3 of 5
SQL Agent Job
  1. Create the RapidReconciler SQL Agent job on the new server using the script from the old server.
  2. Verify the job step points to the correct SSIS package path in the SSISDB catalog (/SSISDB/RapidReconciler/RapidReconciler/RapidReconciler-Prod.dtsx).
  3. Configure the job schedule to match the original server settings.
  4. Do not enable the schedule yet. The schedule stays disabled until the full migration is validated in the cutover step.
4
Step 4 of 5
Install the RR Agent
Set VALC status to “Created” first

The VALC status for this client must be changed to “Created” before installing the agent. If it is not set to “Created”, the agent download prompt will not appear after login.

  1. From the new server, open a web browser and navigate to https://rapidreconciler.getgsi.com.
  2. Log in using the GSIADMIN credentials.
  3. Download and install the RapidReconciler Agent (64-bit in most cases).
  4. Follow the on-screen prompts. The page will display “Installation Complete” when finished.
5
Step 5 of 5
Obtain IP Addresses for VALC
  1. Navigate to the RapidReconciler log file on the new server.
  2. Note the Internal IP address.
  3. Note the External (public-facing) IP address.
  4. Keep both values handy — you’ll need them in the next topic when updating VALC.
Where is the log file?

The log files are always at this exact path on the application server:

C:\Program Files\Rapid Reconciler\logs

The folder contains three log files. The IP addresses you need are in the dated .out.log file (the one that ends in _YYYYMMDD.out.log).

Next

With both IP addresses recorded, continue to VALC & Cloudflare to finalize the cutover.

Separate Servers Migration

Use this procedure when the customer’s RapidReconciler Agent and database are hosted on different machines. The work splits naturally across the two servers — complete the application server work first (it gives you the IP addresses needed for VALC), then the database server work, then move on to the cutover.

Application Server

Software Prerequisites

  • New application server prerequisites
  • SQL Server Management Studio
  • Visual Studio Community Edition with the SQL Server Integration Services extension
  • Applicable JDE database drivers (AS/400 or Oracle)
Note

The RapidReconciler Agent will be installed as part of the migration steps below — it does not need to be pre-installed.

Application Server Process

1
App Server · Step 1 of 4
Copy the SSIS Package
  1. Locate the SSIS package file (.dtsx) on the old application server.
  2. Copy it to a known location on the new application server.
2
App Server · Step 2 of 4
Install the RR Agent
Set VALC status to “Created” first

The VALC status for this client must be changed to “Created” before installing the agent. If it is not set to “Created”, the agent download prompt will not appear after login.

  1. From the new application server, open a web browser and navigate to https://rapidreconciler.getgsi.com.
  2. Log in using the GSIADMIN credentials.
  3. Download and install the RapidReconciler Agent (64-bit in most cases).
  4. Follow the on-screen prompts. The page will display “Installation Complete” when finished.
3
App Server · Step 3 of 4
Obtain IP Addresses
  1. Navigate to the RapidReconciler log file on the new application server.
  2. Note the Internal and External IP addresses — both are required for the VALC update later.
4
App Server · Step 4 of 4
Deploy the SSIS Package
  1. Open Visual Studio Community using the Integration Services project template.
  2. Create a new project named RapidReconciler.
  3. Add the copied .dtsx package to the project.
  4. Update the source (JDE) and destination (RapidReconciler_Prod) connection strings to point to the new servers.
  5. Test all connections to confirm they are valid.
  6. Deploy the package to the SSISDB RapidReconciler folder on the database server.
See Installing the Production Database → Tour 3 for screenshots of each Visual Studio dialog and the full deployment walkthrough.

Database Server

Software Prerequisites

  • New database server prerequisites
  • SQL Server Standard Edition (minimum), Version 2017 or later
  • Administrator access to complete all required tasks
  • Integration Services installed
  • Integration Services Catalog (SSISDB) created
  • RapidReconciler folder added to SSISDB
  • Applicable JDE database drivers (AS/400 or Oracle)
See Installing the Production Database for detailed instructions on creating the SSISDB catalog and the RapidReconciler folder.

Database Server Process

  1. Back up the current production RapidReconciler database on the old server.
  2. Restore the RapidReconciler database on the new database server.
  3. Add the RRUSER SQL Authentication login using the script from the old server.
  4. Create the RapidReconciler SQL Agent job using the script from the old server.
  5. Verify the job step points to the correct SSIS package path in the SSISDB catalog.
Verify before continuing

Run a quick row count on key tables such as F4111 on the restored database and compare against the old server. Confirming counts match before continuing saves significant rework if the restore failed silently.

Next

With both servers prepped and IP addresses recorded, continue to VALC & Cloudflare to finalize the cutover.

VALC & Cloudflare

This topic finalizes the cutover and applies to both migration scenarios. Run it after all server work is finished. The order matters — VALC and Cloudflare must be updated before you enable the SQL Agent job schedule, otherwise the agent has no path back to the application and the first run will fail in confusing ways.

1
Cutover · Step 1 of 5
Update VALC
  1. Navigate to the Client Details page in VALC.
  2. Update both the Internal IP and External IP addresses for the client — use the values you recorded from the new server’s log file.
  3. Navigate to the Databases page in VALC.
  4. Update the Internal IP address.
Double-check the IPs before saving

Confirm the new IP addresses are correct before saving. Entering an incorrect IP will prevent the agent from communicating with VALC and will block the rest of the cutover.

2
Cutover · Step 2 of 5
Update Cloudflare

Contact GSI I/T to update the IP address in Cloudflare. Provide the following information from the Client Details page in VALC:

  • Domain URL (from VALC Client Details)
  • Internal IP (from VALC Client Details)
Wait for DNS propagation

Data will not appear in the RapidReconciler UI until the Cloudflare update is in place. Allow time for DNS propagation after the update before testing.

3
Cutover · Step 3 of 5
Install the RR Agent (if not already done)

If the agent was already installed during the server-specific migration steps, skip this step. Otherwise, install it now from the application server.

VALC status must be “Created”

The VALC status for this client must be set to “Created” before the agent download prompt will appear.

  1. From the application server, open a web browser and navigate to https://rapidreconciler.getgsi.com.
  2. Log in using the GSIADMIN credentials.
  3. Download and install the RapidReconciler Agent.
4
Cutover · Step 4 of 5
Enable the SQL Agent Job Schedule
  1. Confirm the Cloudflare update is in place and data is appearing in the UI.
  2. Enable the SQL Agent job schedule on the new server.
  3. Verify the first scheduled run completes successfully.
5
Cutover · Step 5 of 5
Test Results
  1. Log in to the RapidReconciler UI using the GSIADMIN credentials.
  2. Confirm that data appears correctly in the interface.
  3. Spot-check key figures against the source JDE data to validate accuracy.
  4. Notify the client that the migration is complete and provide any updated connection details they may need.
Next

Walk through the Post-Migration Checklist to confirm every step has been completed before signing off.

Post-Migration Checklist

End-to-end verification list for either migration scenario. Walk this top to bottom before signing off the migration. Anything unchecked is a blocker for go-live.

Pre-Migration

  • Before starting
  • Client notified of maintenance window

Database Work

  • Database
  • Database backed up from old server
  • Database restored on new server and record counts verified
  • RRUSER login created on new server
  • SSISDB catalog and RapidReconciler folder created (if new install)

Application & SSIS Work

  • Application and SSIS
  • SSIS package copied, updated, and deployed
  • SQL Agent job created and configured (schedule not yet enabled)
  • RR Agent installed on server
  • Internal and External IP addresses noted from log file

Cutover

  • VALC and Cloudflare
  • VALC Client Details updated with new IP addresses
  • VALC Databases page updated with new Internal IP
  • Cloudflare updated by GSI I/T
  • DNS propagation confirmed

Validation & Sign-Off

  • Validation
  • SQL Agent job schedule enabled
  • First job run completed successfully
  • Data verified in RapidReconciler UI
  • Client notified that migration is complete
Done?

Every box checked? Migration is complete — archive the old server backup per your retention policy, and document any client-specific notes that emerged during cutover for the next time.
