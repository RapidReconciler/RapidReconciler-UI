Agent download prompt doesn’t appear after login

Network tech logs in to rapidreconciler.getgsi.com with gsiadmin credentials but no agent download prompt is shown. Affects fresh installs and migrations — the underlying causes overlap, with one extra check for migrations (existing agent registration on the old server interfering).

What you’re seeing

  • Page loads normally after login but the agent download UI is missing.
  • Affects both fresh installs and migrations to a new server.
  • The Setup Step value on Client Details usually controls whether the prompt renders.

Walk these checks

Check 1
Verify Setup Step is “Created” in VALC

The agent download prompt only renders when the customer’s Setup Step is set to Created on the Client Details tab. Any other value hides the prompt.

  1. In VALC, open Manage Client → Client Details for the customer.
  2. Verify Setup Step is Created. If not, change it to Created and save.
  3. Have the customer hard-refresh the browser (Ctrl+F5 / Cmd+Shift+R) and log in again with gsiadmin credentials.
Resolved? Stop here. Otherwise, continue to Check 2.
Check 2
Migration only — clear old-server agent registration
When this applies: migrating to a new server.

Existing agent registration on the old server may still be active and prevent the prompt from appearing on the new server.

  1. In VALC, set the Setup Step on the migrating customer to Created and save.
  2. Hard-refresh the browser on the new server and log in again with gsiadmin credentials.
  3. The agent download prompt should now appear.
Resolved? Stop here. Otherwise, confirm no other browser session has the customer logged in elsewhere, then escalate.
