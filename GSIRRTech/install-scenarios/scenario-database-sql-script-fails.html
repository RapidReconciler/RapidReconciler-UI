SQL script fails during database install

One of the four install scripts (database creation, database objects, rruser creation, SQL Agent job) errors out partway through. Each script has a distinct set of common causes — identify which one is failing, then walk the checks for that script.

What you’re seeing

  • SSMS shows an error after executing one of the four install scripts shipped in the install zip.
  • Subsequent scripts can’t run because their prerequisites aren’t in place (e.g. script 2 fails because script 1 didn’t finish, or script 4 fails because the SQL Agent service isn’t running).
  • Common error wordings: CREATE FILE encountered operating system error, Cannot find object, Login failed, Password validation failed, SQL Server Agent service not running.

Walk these checks

Check 1
Identify which script failed

The install zip ships four scripts that must run in order:

  1. 1 - RapidReconciler Database Creation Script.sql
  2. 2 - RapidReconciler Database Object Script 178.sql
  3. 3 - RapidReconciler User Creation Script.sql
  4. 4 - RapidReconciler SQL Agent Job Creation Script.sql

Note the exact error text and which script produced it — the rest of the checks branch on the script number.

Identified? Jump to the matching check below.
Check 2
Script 1 (Database Creation) failed

Common causes:

  • CREATE FILE encountered operating system error — the SQL Server instance’s default data path doesn’t exist on this server, or the SQL Server service account lacks write permission to the drive. Confirm the instance’s default data and log paths in SSMS (right-click instance → Properties → Database Settings) and that the SQL Server service account can write to them.
  • Database ‘RapidReconciler_Prod’ already exists — a prior failed install attempt left a stub database. Confirm with the customer DBA that the database is safe to drop, then DROP DATABASE RapidReconciler_Prod and re-run script 1.
  • Login failed for user — SSMS is connected with credentials that don’t have permission to create databases on this instance. Reconnect with an account that has sysadmin or dbcreator.
Script 1 succeeded? Continue with script 2.
Check 3
Script 2 (Database Objects) failed

Common causes:

  • Cannot find object or Invalid object name ‘RapidReconciler_Prod…’ — script 1 didn’t finish cleanly. Verify the database exists and has the expected file groups before re-running script 2.
  • Syntax errors on T-SQL features — the script targets SQL Server 2016 or newer. Confirm the target instance’s version with SELECT @@VERSION; older instances aren’t supported.
  • Partial run leaves orphan objects — if the script failed midway through, drop the database with DROP DATABASE RapidReconciler_Prod and re-run scripts 1 and 2 from a clean state rather than re-running script 2 on top.
Script 2 succeeded? Continue with script 3.
Check 4
Script 3 (rruser Creation) failed

Common causes:

  • Login ‘rruser’ already exists — a prior attempt left the login behind. Drop it with DROP LOGIN rruser and re-run.
  • Password validation failed / Password does not meet Windows policy requirements — the instance has a stricter password policy than the default the script uses. Either temporarily relax the policy on the SQL Server, or open the script and replace the password with one that meets the customer’s policy (and record the new value — you’ll need it when configuring the RRLocal connection manager in the SSIS package).
  • Mixed Mode authentication not enabled — SQL Authentication logins won’t work if the instance is Windows-Authentication-only. Enable Mixed Mode (right-click instance → Properties → Security) and restart the SQL Server service.
Script 3 succeeded? Continue with script 4.
Check 5
Script 4 (SQL Agent Job) failed

Common causes:

  • SQL Server Agent service is not running or SQLServerAgent is not currently running — start the SQL Server Agent service via the Services console (services.msc) or SQL Server Configuration Manager. Set its start mode to Automatic so it survives reboots.
  • The specified ‘@job_name’ is already used — a prior attempt left the job behind. Drop it from SSMS (SQL Server Agent → Jobs → right-click RapidReconciler_Prod → Delete), or run EXEC msdb.dbo.sp_delete_job @job_name = 'RapidReconciler_Prod', then re-run script 4.
  • SQL Agent service account lacks sysadmin — required to create jobs in msdb. Verify the service account’s permissions; if locked down, run the script as a Windows account that does have sysadmin.
Script 4 succeeded? Database install is complete — continue to Step 8 (SSIS configuration).
Check 6
Still stuck

If a script keeps failing after the targeted checks above, escalate to the GSI DBA with:

  • The exact error text from SSMS (copy-paste, including the line number).
  • Which script (1, 2, 3, or 4) is failing.
  • The SQL Server version (SELECT @@VERSION) and edition.
  • Whether prior install attempts on this database have happened — leftover state is the most common compounding cause.
Escalated? Do not proceed to Step 8 until the database is fully created.
