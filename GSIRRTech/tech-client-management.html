Welcome. This is the five-step client-install process — the work that begins when Customer I/T's pre-install email lands in rrsupport@getgsi.com and ends when the customer goes live. The chart on the right is the master index.

How to use this page

  • Step 3 is shared — a 2-hour live working session between RR DBA and Customer I/T.
  • Step 5 starts only after the DBA's completion email lands in rrsupport@getgsi.com.

Step 1

Receive install prep email

RR DBA

RR Support routes the customer's pre-install submission email to RR DBA, with subject "RapidReconciler Pre-Install Submission — {Customer Name}". This is the trigger that starts the technical install. RR DBA reviews the submitted topology, source platform, and server details before scheduling the install meeting. If a PoC was previously run for this customer, the production install rebuilds from the PoC environment rather than starting clean.

Step 2

Schedule the install meeting

RR DBA · coordination

RR DBA schedules the 2-hour live web meeting with Customer I/T using the standard 9-item agenda below. Customer I/T must be available for the full duration with local administrator credentials in hand. Multi-night installs are not supported.

Standard 9-item agenda

  1. Architecture & topology review

    Confirm Configuration 1, 2, or 3; verify hardware, RAM, disk-space minimums; confirm OLE DB provider architecture (32-bit and/or 64-bit per JDE source).

  2. Download install zip on screen-share

    DBA downloads a fresh install zip directly to the customer's database server and unzips it — eliminates the most common "missing files" failure mode.

  3. Create SSISDB catalog

    Set up the Integration Services Catalog and the RapidReconciler folder on the target SQL Server.

  4. Install RapidReconciler database

    Run the four installation scripts: create the database, schema objects, the rruser SQL login, and the RapidReconciler_Prod SQL Agent job.

  5. Configure & deploy SSIS package

    Configure JDE source and RR destination connection managers, set SSIS variables from the submission, deploy to the catalog, and point the SQL Agent job at the deployed package.

  6. Schedule initial data load

    Enable the SQL Agent job's schedule against an off-peak run time agreed with Customer I/T.

  7. Configure security & permissions

    Verify rruser permissions (db_datareader, db_datawriter, SQLAgentOperatorRole on msdb), SQL Agent service-account access, and SSIS catalog roles.

  8. Customer-side load verification

    Customer I/T confirms the first scheduled run completed successfully via emailed confirmation. This is the gate before the completion email goes out.

  9. Wrap-up & Network Tech handoff

    Send the database-ready completion email to rrsupport@getgsi.com; preview the separate VALC working session that comes next with RR Network Tech.

Step 3

Configure RR Database & SSIS Package

RR DBA leads · Customer I/T provides server access & JDE details

The 2-hour shared working session. RR DBA drives; Customer I/T provides server access, JDE source platform credentials, and any environment-specific details. Covers SSISDB catalog setup, RR database creation scripts, SSIS package configuration and deployment, and initial data load. The session ends when the load has been verified.

Step 4

Submit completion email to RR Support

RR DBA → RR Support

RR DBA emails rrsupport@getgsi.com with subject "RapidReconciler Database Ready — {Customer Name} — VALC Install Can Begin". Body includes customer name, SQL port (only if non-default 1433), rruser password (only if non-default), confirmation the load was verified, and a topology-filtered pre-flight checklist. This email is the handoff out of the DBA workflow and into the Network Tech workflow — VALC install scheduling starts at this point.

Step 5

Install client in VALC

RR Network Tech

RR Network Tech runs the 13-step VALC procedure end-to-end: client registration, DNS A-record & certificate request, agent install (a separate working session with Customer I/T), SQL Server validation, database registration, license configuration, and customer go-live handoff with login URL and initial credentials. After this completes, the customer is live.
