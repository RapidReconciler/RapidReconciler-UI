CASE 003 2025-03-07

Post-deployment JDE import fails — rruser login error

Customer-side admin deployed RR to production successfully but Import JDE data errors out after a few minutes with an rruser authentication failure. The first email-based fix — "update your connection passwords" — was correct in principle but pointed at the wrong layer; the customer fixed the deployment-time connections and missed the package-internal one. Resolved in a 15-minute live Teams session: open the SSIS package in Visual Studio, find the rruser Connection Manager, re-enter the password, redeploy.

CASE 004 2026-02-23

System-wide “no data” — DNS filtering on customer side

Customer-admin reports an urgent morning outage — all users, no data, admin panel says “no database connection.” System worked the previous afternoon. Customer had pre-emptively ruled out Citrix, JDE CNC, and certs. The diagnostic gold was the external-vs-internal DNS test: nslookup and ping failed from inside the customer's network, succeeded from outside. Root cause was a DNS-filtering rule the customer had added two days earlier that inadvertently blocked *.getgsi.com. Customer's IT removed the rule; system back online.

CASE 005 2026-01-12

Blank screen, single user — browser blocking local network access

Single user reports a blank screen on RR — page loads but no data shows. Colleagues fine. Walking the standard scope-first funnel (single-user check, cross-machine test, ping, basic browser hygiene) eliminated network and server causes; the matched scenario card’s prescription — enable the browser’s Local Network Access permission for the RR site — resolved it. First case in the library where a scenario card matched the symptom. Validates the matcher’s existing card library on this class of issue.
